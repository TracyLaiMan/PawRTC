1. 噪声类型与 ANS 在音频链路中的位置

稳态噪声 vs 非稳态噪声 · ANS 位于 AEC 之后、AGC 之前 · STFT 频域处理框架

噪声分类

不同噪声需要不同的处理策略

稳态噪声 (Stationary): 风扇声、空调声、道路白噪声、电路底噪 — 频谱特性不随时间变化。谱减法/Wiener 可有效处理。
非稳态噪声 (Non-stationary): 键盘敲击、关门声、狗叫、餐具碰撞 — 频谱特性瞬态变化。传统方法难以处理 → 需要深度学习 (RNNoise/PercepNet)。
混响 (Reverberation): 房间反射 — 不同于加性噪声，需要去混响 (Dereverberation) 专门处理。

ANS 在音频 Pipeline 中的位置

Mic → HPF → AEC → ★ ANS → AGC → Encoder

AEC 之后: 回声消除必须先于降噪 — 否则 ANS 会把回声当作噪声学习 → 模型污染。
AGC 之前: 降噪后再增益 — 如果先增益会把噪声也放大 → 降低信噪比。
处理框架: 大多数 ANS 在频域 (STFT) 操作 — 10ms 帧, 50% 重叠, Hann/Hamming 窗。核心操作: 估计噪声谱 → 计算增益 → 乘到幅度谱 → 保留相位 → ISTFT。

Gain Computation — 根据 |Y(ω)| 和 |N̂(ω)| 计算每频点的衰减增益 G(ω) │ ├── 谱减法: G = max(|Y|² - β|N̂|², γ·|Y|²) / |Y|² │ ├── Wiener: G = ξ/(1+ξ) (ξ = a priori SNR) │ └── DNN: G = sigmoid(network_output) │ ├── 5. 2. 谱减法 (Spectral Subtraction) — 1979 年经典

最直观的降噪方法 · |Ŝ|² = |Y|² - |N̂|² · 过减因子 β · 谱下限 γ · 音乐噪声的产生与抑制

基本原理

噪声功率谱减法 — 最直观的思路

核心公式: |Ŝ(ω)|² = |Y(ω)|² - |N̂(ω)|²
假设: 噪声是加性且与语音不相关 → 功率谱可加。噪声是稳态的 — 静音段估计的噪声谱可代表全段。
噪声估计: 在静音段 (VAD=0) 对 |Y|² 做指数平均: |N̂|² = α·|N̂_old|² + (1-α)·|Y|²。
致命缺陷: 减法后可能出现负值 → 设下限为 0 → 产生随机孤立频谱峰值 → 听感上像"音乐噪声" (随机音调)。

过减 + 谱下限 — 缓解音乐噪声

β 控制过减 · γ 控制下限 · α 控制平滑

改进公式: G = max( (|Y|² - β·|N̂|²) / |Y|², γ )
β (过减因子, 1.5~3.0): 加倍扣除噪声 → 残留噪声幅度降低，但语音失真增加。
γ (谱下限, 0.002~0.2 ≈ -27dB~-14dB): 保证所有频点至少保留 γ 倍原幅度 → 避免"静音"的孤立峰值（音乐噪声的根因）。
自适应 β: 高 SNR 段 → 低 β (1.5) · 低 SNR 段 → 高 β (2.5~3.0) → 平衡失真与残留噪声。

3. Wiener 滤波 — 最优线性估计

频域最小均方误差最优解 · 决策导向 SNR 估计 (Ephraim-Malah 1984) · 理解它是理解一切现代方法的基石

Wiener 增益函数

G = ξ / (1 + ξ) — 极其简洁

推导: 令 E[|Ŝ - S|²] 最小 → 频域解: G(ω) = ξ(ω) / (1 + ξ(ω))
其中 ξ = P_s / P_n = 先验 SNR (a priori SNR)。
与谱减法的关系: 当用瞬时 SNR 估计先验 SNR 时, Wiener 退化为谱减法。
关键挑战: 先验 SNR ξ 是未知的 — 我们需要从含噪语音中估计它。这就是 Ephraim-Malah 1984 的核心贡献。

决策导向 (DD) SNR 估计 ★

ξ̂ = α·Ŝ²_prev/N̂ + (1-α)·max(γ-1, 0)

后验 SNR: γ = |Y|² / |N̂|² — 可以直接算 (已知 |Y|, 估计 |N̂|)。
先验 SNR (DD 估计): ξ̂(n) = α·|Ŝ(n-1)|²/|N̂|² + (1-α)·max(γ-1, 0)
α (平滑系数, 默认 0.98): 第一项 = 上帧的干净语音功率 / 当前噪声 (利用帧间相关性) · 第二项 = 当前观测的瞬时 SNR (快响应)。
为什么 DD 比直接用后验 SNR 好? 后验 SNR 逐帧波动剧烈 → 增益剧烈波动 → 音乐噪声。DD 的平滑作用大幅抑制这种波动。这是现代降噪算法的基石。

4. MMSE 估计器 — 超越线性

MMSE-STSA · MMSE-LSA · 复杂的非线性增益公式 · 音乐噪声几乎消除 · 画质最好的传统方法

MMSE-STSA (Ephraim-Malah 1984)

最小均方误差 — 短时谱幅度

假设: 语音和噪声的 DFT 系数都服从复高斯分布 → 在此模型下推导最优幅度估计器。
增益: 不是简单的闭式解 — 涉及 Bessel 函数 (I₀/I₁) 的复杂表达式。计算机查表实现。
效果: 比 Wiener 滤波的音乐噪声显著减少 — 因为 Bessel 函数自然提供了"软掩膜"效果。

MMSE-LSA (Ephraim-Malah 1985)

对数谱幅度 — 感知更优

改进动机: 人耳对对数幅度 (dB) 而非线性幅度敏感。
目标: E[(log|S| - log|Ŝ|)²] 最小化。
增益: G_LSA = (ξ/(1+ξ)) · exp(½∫(e^{-t}/t)dt) — 含指数积分。
效果: 比 MMSE-STSA 的感知质量更好 — 是传统方法的天花板 (在深度学习出现之前)。WebRTC ANS 的增益计算部分基于 LSA 思想。

5. WebRTC ANS 源码深度 — ns_core.c

分位数噪声估计 · 16 子带差异化处理 · 3 特征语音/噪声分类 · 三级平滑消除音乐噪声

/* ===== WebRTC ns_core.c — 核心算法伪代码 ===== */ /* 输入: 10ms 帧, 16kHz, FFT=256点 → 129 频点 → 16 子带 */ // ===== 1. 噪声估计: 分位数跟踪 (Quantile Noise Estimation) ===== void NoiseEstimation(float magn[129], float noise[129]) { for (int i = 0; i < 129; i++) { float lmagn = log(magn[i] + 1e-10); // 对数域处理 // 分位数更新: 若当前幅度 > 分位数 → 分位数缓慢上升 if (lmagn > lquantile[i]) { lquantile[i] += 0.001; // 上升慢 (适应噪声增大) } else { lquantile[i] -= 0.01; // 下降快 (跟踪噪声减小) } noise[i] = exp(lquantile[i]); // 回线性域 } } // ===== 2. 语音/噪声分类: 3 特征 + 直方图 ===== float SpeechProbability() { float lrt = ComputeLRT(); // 似然比 — 频谱形状匹配度 float flat = SpectralFlatness(); // 频谱平坦度 — 噪声平坦 vs 语音谐波 float diff = SpectralDifference(); // 与噪声模板的距离 // 每个特征的直方图 (500 帧) → 自适应阈值 → 概率映射 return w_lrt * P_lrt + w_flat * P_flat + w_diff * P_diff; } // ===== 3. 子带增益计算: SNR → 自适应 β,γ ===== float ComputeGain(float signal, float noise, float snr) { float beta = 1.5; float gamma = 0.01; // 默认值 if (snr < 0.5) { beta = 3.0; gamma = 0.002; } // 低 SNR → 激进降噪 else if (snr > 2.0) { beta = 1.5; gamma = 0.05; } // 高 SNR → 保守降噪 float wiener_gain = prior_snr / (1 + prior_snr); float ss_gain = max((signal - beta*noise)/signal, gamma); return min(wiener_gain, ss_gain); // 取两者更保守 } // ===== 4. 三级平滑 — 消灭音乐噪声 ===== float SmoothGain(float gain, float prev_gain, int bin) { // L1: 频谱平滑 — 5-point 中值滤波 (相邻频点) gain = median5(gain[bin-2], ..., gain[bin+2]); // L2: 时间平滑 — IIR 一阶滤波 float alpha = (snr[bin] > 1) ? 0.9 : 0.7; // 语音段平滑更多 gain = alpha * prev_gain + (1-alpha) * gain; // L3: 增益下限 — 保证不丢语音 return max(gain, 0.001); // -30dB floor }

WebRTC ANS 的 16 子带策略

不同频段的差异化处理

0–2 kHz (低频): 最激进降噪 — 这里噪声最集中 (风扇/空调/道路)。
2–4 kHz (中频): 平衡处理 — 语音共振峰在此 (F1/F2) → 保护语音清晰度。
4–8 kHz (高频): 最保守降噪 — 保留环境自然感，避免"真空感"。
子带间通过三次样条插值平滑过渡 → 避免子带边界产生不连续。

WebRTC ANS 版本演进

三代: 固定阈值 → 统计模型 → 深度学习

Gen 1 (已废弃): 固定阈值谱减法。
Gen 2 (当前 C 实现): 分位数跟踪 + Wiener/MMSE 混合 + 子带处理。
Gen 3 (M94+, 当前推荐): ① PercepNet — 结合 DSP 与轻量 DNN (感知优化) ② NSNet2 — RNN 基模型。
API: config.noise_suppression.level = kHigh (Low/Moderate/High/VeryHigh)

GRU 核心方程

门控循环单元 — LSTM 的简化版

为什么用 GRU 而非 LSTM? 参数量少 25%, 计算量更低, 降噪任务性能相当。
四个核心公式:
z = σ(W_z·x + U_z·h) — 更新门 (控制保留多少过去状态)
r = σ(W_r·x + U_r·h) — 重置门 (控制用多少过去状态计算候选)
h̃ = tanh(W_h·x + U_h·(r⊙h)) — 候选新状态
h_new = z⊙h + (1-z)⊙h̃ — 新隐藏状态 (z 接近 1 → 保留旧状态; 接近 0 → 接受新候选)

训练与部署

PyTorch 训练 → C 部署 · 0.5MB · 10ms

训练数据: 干净语音 (TTS) + 噪声库 (DEMAND/街道/人群) × 混合 SNR (−5~+20dB) + 混响 (RIR)
损失函数: L = mean((1+5·VAD)·mask·(ĝ^0.25 - g^0.25)²) — 语音段加权 6×, γ=0.25 感知指数
部署: PyTorch → C 数组 (权重硬编码) → SSE4.1/AVX2 运行时自动选择。
2024 新增: rnnoise_model_from_file() 加载外部 .rnnn 模型 — 可针对特定场景训练定制模型。
