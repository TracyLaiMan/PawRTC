区域感知

CQP / CRF

从固定 QP 到感知恒定质量

CQP: 所有帧同一 QP — 简单但低效。CRF: x264/x265 的默认模式 — mb-tree + 自适应量化 使高运动帧 QP 更高 (人眼不敏感)，平坦帧 QP 更低。

CBR / VBR + VBV

比特流约束下的最优分配

CBR: VBV 缓冲模型严格限制码率波动 — 直播/RTMP 必须。VBR: 允许码率波动，在目标平均码率下最大化质量。VBV 是 CBR 的核心 — 模拟解码器缓冲区, 防止上溢(丢帧)和下溢(填充)。

ROI 编码

QP delta map — 差异化质量分配

对人脸/运动区域分配更低的 QP (高质量)，背景/静态区域分配更高的 QP (低质量)。通过 QP delta map (每 MB/CTU 一个 ΔQP) 实现。x264/x265/VVC 均支持。

1. CQP — 恒定量化参数

所有帧使用相同的 QP · 最简单的码率控制 · 码率完全不可预测 · 通常仅用于测试和全 I 帧场景

工作原理

Q_step = f(QP) — 查表映射, 所有帧相同

CQP = Constant Quantization Parameter。编码器不对码率做任何控制 — 只按固定的 QP 值量化每个宏块/CTU。
Q_step 与 QP 的关系: H.264/265 中, QP 每增加 6, Q_step 翻倍。QP=0 表示无损 (H.264 High 4:4:4), QP=51 表示最大压缩。
码率不可预测: 同样 QP=23, 静态场景可能 500 kbps, 高速运动场景可能 50 Mbps — 完全取决于画面复杂度。

适用场景 & 限制

仅测试/中间格式 — 不适合传输

适用: ① 编码器 benchmark (对比不同编码器的压缩效率) ② 视频编辑中间格式 (全 I 帧) ③ 快速验证某个 QP 下的画质。
不适用: 任何需要控制码率的场景 — 直播 (码率不可控 → 带宽溢出) · 存储 (空间不可预测) · 网络传输。
CQP vs 固定 bitrate: QP=18 可能 5~30 Mbps — 取决于画面。x264 CLI: --qp 23

2. CRF — 恒定速率因子 (x264/x265 默认)

基于感知模型的自适应 QP — mb-tree 传播分析 · 自适应量化 · qcomp 曲线压缩 · 非恒定 QP!

mb-tree — 宏块树

QP 帧内分配的核心算法

原理: 对每 MB 计算其被后续帧参考的频率和强度。被大量参考 (信息传播到多帧) 的 MB → 降低 QP (更多 bits) — 因为它的质量影响后续多帧。不被参考的 MB (背景/一次性出现) → 升高 QP (更少 bits)。
lookahead: 需要分析未来 N 帧 (rc-lookahead, 默认 40)。
mb-tree 的 QP 调整: QP_adjusted = QP_base + 8×(1 - propagation_cost)

自适应量化 (AQ)

帧内 MB 级别的 QP 微调

AQ Mode 1 (默认): 基于空间复杂度 — 平坦区域 (天空/墙壁) 降 QP (更多 bits, 避免色带) · 纹理复杂区域升 QP (人眼不敏感)。
AQ Mode 2 (auto-variance): 基于方差 — 更激进地提升平坦区域质量。
AQ strength: 0.0~2.0, 默认 1.0。越大 → 平坦区域质量越高, 纹理区域质量越低。

qcomp — QP 曲线压缩

控制帧间 QP 波动幅度

qcomp (0~1, 默认 0.6): 控制码率-质量曲线的"压缩度"。
qcomp=0: CBR 模式 — 所有帧 QP 非常接近 (帧间波动小)。
qcomp=1: CQP 模式 — 帧间 QP 差异最大化 (恒定 Qscale)。
qcomp=0.6: 平衡 — 高运动帧适度增 QP, 静态帧适度减 QP。
调优: 降低 qcomp (如 0.5) → 运动场景码率更平滑, 适合直播。

/* ═══ CRF 的 QP 决策流程 — 每帧/每MB 三层决策 ═══ */ Level 1: 帧级 crf QP rate_factor_constant = -qcomp * log2(current_satd / average_satd) frame_qp = crf_value + rate_factor_constant // satd = 帧的 SATD 复杂度 (Sum of Absolute Transformed Differences) // 高复杂度帧 → 正 offset → 更高 QP Level 2: mb-tree 调整 (每 MB) propagation_cost = Σ{后续帧中该 MB 被引用的权重} mb_qp_offset = 8.0 * (1.0 - propagation_cost) // propagation_cost 高 → 负 offset → 更低 QP (该 MB 被大量后续帧参考) Level 3: AQ 微调 (每 MB) spatial_complexity = variance(MB) mb_qp += aq_strength * log2(spatial_complexity / avg_complexity) // 平坦 MB → 负 offset → 更低 QP (人眼对平坦区域色带敏感) 最终 QP = crf_qp + mb_tree_offset + aq_offset

3. CBR — 恒定码率 (VBV 缓冲模型)

直播/实时通信的标配 · VBV 模拟解码器缓冲区 · 上溢(填充 → 空帧)和下溢(消耗 → QP 增大)机制

VBV 缓冲模型 — CBR 的心脏

VBV = Video Buffering Verifier

模型: 模拟一个固定大小的解码器缓冲区。编码器每帧写入 frame_bits, 解码器以恒定速率 bitrate/fps 消耗。
上溢 (overflow): 缓冲区填满 → 编码器不能再写入更多 bits → 必须填充空数据 (stuffing bits) 或增大 QP。
下溢 (underflow): 缓冲区为空 → 解码器无数据可取 → 画面卡顿 → 编码器必须降低 QP 增加填充。
VBV buffer size: 默认 = bitrate (1 秒缓冲)。网络直播通常设 0.5~2 秒。

CBR 的 QP 调整循环

每帧编码后评估 VBV 状态 → 调整下一帧 QP

① 目标码率 target_bitrate → 每帧预期 bits = target_bitrate / fps
② 编码当前帧 → 实际 bits = frame_bits
③ 充盈度 = 前一充盈度 + frame_bits - expected_bits_per_frame
④ 若充盈度 > buffer_size × 0.9 (近满) → 下帧 QP 大幅增加 (速率惩罚)
⑤ 若充盈度 < buffer_size × 0.1 (近空) → 下帧 QP 降低 (加快填充)
⑥ 重复②~⑤, 使长期平均码率 = target_bitrate

/* ===== CBR 内部算法伪代码 (x264 ratecontrol.c) ===== */ void rate_control_cbr(x264_t *h, int frame_bits) { double expected = h->param.rc.i_bitrate / h->param.i_fps_num; double overflow = 1.0; // QP 缩放因子 // 1. 更新 VBV 充盈度 h->rc.buffer_fill += frame_bits - expected; if (h->rc.buffer_fill < 0) h->rc.buffer_fill = 0; // 2. 上溢检查 — 缓冲区近满 → 大幅增 QP if (h->rc.buffer_fill > h->param.rc.i_vbv_buffer_size * 0.9) { double extra = h->rc.buffer_fill / (h->param.rc.i_vbv_buffer_size * 0.1); overflow = pow(2.0, extra / 6.0); // 指数惩罚 — QP 每+6 码率减半 } // 3. 下溢检查 — 缓冲区近空 → 减 QP 加速填充 if (h->rc.buffer_fill < h->param.rc.i_vbv_buffer_size * 0.1) { overflow *= 0.5; // QP 减半 → 码率加倍 } // 4. 应用 QP 调整 h->rc.qpa_rc += overflow; h->rc.qpa_rc = CLIP3(0, 51, h->rc.qpa_rc); }

4. VBR — 可变码率

在目标平均码率下最大化质量 · 允许瞬时码率波动 · 适合点播/存储 — 不适合实时直播

VBR 核心思想

让编码器自由分配 bits — 简单场景少用, 复杂场景多用

VBR = Variable Bitrate。与 CBR 不同, VBR 不强制每帧/每秒码率恒定 — 它只保证整个视频的平均码率 = target_bitrate。
策略: ① 两遍编码 (2-pass): 第一遍分析复杂度 → 生成统计文件 → 第二遍根据统计分配 bits ② 单遍 VBR: 通过 crf + vbv-maxrate + vbv-bufsize 实现 CRF 限制上限的 VBR。
优势: 同等平均码率下, VBR 画质远优于 CBR (简单场景不浪费, 复杂场景不压缩过度)。

受限 VBR (capped VBR)

CRF + VBV-maxrate = 带码率上限的高质量编码

实现: --crf 23 --vbv-maxrate 4000 --vbv-bufsize 8000
CRF 控制质量 → VBV maxrate 限制瞬时码率上限 → 缓冲吸收波动。
典型配置: vbv-bufsize = 2 × vbv-maxrate (2 秒缓冲)。
这比纯 CBR 画质好很多 — 因为只在必要时才限制码率 (复杂场景), 简单场景仍享低码率。

模式QP 决策码率波动延迟质量 (同码率)适用
CQP固定 QP极大最低—测试/Benchmark
CRFmb-tree + AQ 自适应大低最佳存档/离线编码
CBRVBV 缓冲模型驱动极小低最差直播/RTMP/WebRTC
2-Pass VBR第一遍分析→第二遍最优分配中高 (2遍)最佳 (同码率)点播/存储
CRF+VBRCRF + VBV maxrate 上限中中好直播(高质量)

5. ROI 编码 — 差异化质量分配

QP delta map 逐块控制 · 人脸/运动降 QP · 背景升 QP · 感知画质不变, 码率降低 20-40%

QP Delta Map — 核心机制

per-MB/per-CTU 的 QP 偏移量数组

公式: QP_block = QP_rc + delta_qp[i]
delta_qp: 每 MB/CTU 一个值。负值 → 更低 QP → 更高质量 (更多 bits)。正值 → 更高 QP → 更低质量 (更少 bits)。
范围: 通常 -51 ~ +51 (但极端值可能破坏码控)。
粒度: H.264 = 16×16 MB · H.265 = 32×32 sub-CTU · VVC = CTU。
码率中性: 用 ROI_QP_map[i] - ROI_AVG_QP 做偏移 → 总码率基本不变, 只是重新分配。

典型应用场景

人脸视频通话 · 车牌识别 · 工业检测

视频通话: 人脸区域 ΔQP=-6 (高质量) + 背景 ΔQP=+6 (低质量) → 人脸清晰度不变, 码率降低 20-40%。
车牌识别 IPC: 画面中心 (车牌) ΔQP=-8, 边缘 ΔQP=+4 → 确保车牌可识别。
游戏/桌面共享: 鼠标/光标区域 ΔQP=-4 → 文字清晰。
实现途径: ① FFmpeg: AVRegionOfInterest side data · ② x265: --roi 或 API x265_picture_t.roi · ③ V4L2: V4L2_CID_MPEG_VIDEO_ROI_MODE + QP map · ④ VA-API: VAEncMiscParameterBufferROI

/* ===== FFmpeg ROI — 人脸区域高质量编码 ===== */ # 方法1: FFmpeg addroi filter — 矩形ROI ffmpeg -i input.mp4 -vf "addroi=iw/4:ih/4:iw/2:ih/2:-0.5" \ -c:v libx264 -crf 23 output.mp4 # iw/4:ih/4 = ROI 起始坐标, iw/2:ih/2 = ROI 宽高, -0.5 = qoffset # qoffset 范围 -1.0 ~ +1.0 (负值=更高质量) /* ===== x265 API — QP delta map (per-CTU) ===== */ #include <x265.h> // 1. 分配 QP delta 数组 (每 CTU 一个值) int ctu_count = (width/64) * (height/64); // 64×64 CTU int8_t *qp_delta = calloc(ctu_count, sizeof(int8_t)); // 2. 填充 QP delta: 人脸区域 = -6, 背景 = +4 for (i = 0; i < ctu_count; i++) { if (is_face_region(i)) qp_delta[i] = -6; // 更高质量 else qp_delta[i] = 4; // 更低质量 } // 3. 编码每帧时设置 x265_picture_t pic; pic.quantOffsets = qp_delta; // ★ QP delta map x265_encoder_encode(encoder, &nal, &nal_count, &pic, &out_pic); /* ===== V4L2 ROI (Linux 内核, 2024 新增) ===== */ struct v4l2_ctrl_vp9_frame roi; // V4L2_CID_MPEG_VIDEO_ROI_MODE — 矩形或 QP map // V4L2_CID_MPEG_VIDEO_ROI_BLOCK_SIZE — 16x16 / 32x32 / 64x64 // QP map: uint8_t 数组, 栅格扫描序, 每 block 一个值 ioctl(fd, VIDIOC_S_EXT_CTRLS, &ctrls); // 设置 roi QP map
🔧 ROI 实战效果 — 数据驱动
典型视频通话: 人脸占画面 30%, ΔQP_face=-6, ΔQP_bg=+6 → 码率降低 25-35%, 主观画质不变。
车牌识别: 车牌区域占画面 10%, ΔQP=-10 → 车牌清晰度提升相当于整体 QP 降低 4, 整体码率仅增加 5%。
关键: Delta QP 的码率中性设计 (QP_block = QP_rc + (delta_qp[i] - avg_delta)) — 负 delta 从正 delta 回收 bits → 总码率不变。
