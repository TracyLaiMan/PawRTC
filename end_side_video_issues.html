成因分析

1. 光子散粒噪声：光子到达是泊松过程，信号S的噪声=√S。暗光信号弱 → SNR低。

2. 读出噪声：ADC量化+热噪声，与信号无关的固定底噪。

3. 高增益放大：ISO↑ → 增益↑ → 噪声一同放大。小像素尤其严重。

4. 3DNR不足：运动区域跳过时域滤波，只靠空域 → 噪声粗糙且带彩色。

ISO增益15 3DNR2

修复方案

ISP 层：

• 加强3DNR时域降噪权重，运动区域自适应

• 空域降噪用双边滤波保边缘

• 暗区单独提亮+降噪

编码层：

• 编码前做 pre-denoise

• 提高MinQP避免噪声被过度编码

硬件层：

• 更换大像素传感器（IMX335 vs IMX307）

• 补光红外LED改善夜间SNR

IMX307 夜间噪声大是公认问题，3DNR调参是主要手段。

成因分析

AWB不收敛：灰色世界假设在单色场景下失效，色温估计漂移。

IR-Cut时序：切换时红外进入 → 红色偏重。延迟越大偏色越重。

• 肤色优先策略，避免人物偏绿/偏蓝

• 提前切换IR-Cut：lux阈值8-10

• 切换期间冻结AWB，等稳定后恢复

IDR间隔

成因分析

H.264以16×16宏块独立变换量化。相邻宏块量化步长不同 → 边界像素跳变。QP每+6步长翻倍。

QP38

修复

1. 提高码率（直接有效）

2. 限制MaxQP（保底）

3. VBR模式（灵活分配）

4. Deblocking参数
alpha_c0=-2

MaxQP=36是板端常用安全线。

代码

stEncChnAttr.stRcAttr.enRcMode =
  IMP_ENC_RC_MODE_VBR;
stEncChnAttr.stRcAttr.stVbrAttr
  .u32MaxBitRate = 2500;
stEncChnAttr.stRcAttr.stVbrAttr
  .u32MaxQp = 36;
param->deblocking = 1;
param->i_deblocking_alpha_c0=-2;

成因分析

锐利边缘=高频信号。4×4 DCT量化截断高频 → Gibbs现象：边缘旁过冲和振铃。

⚠ 振铃不是块效应！Deblocking只作用于宏块边界，对振铃完全无效。

区分：
• 块效应：沿16×16网格线
• 振铃：沿真实图像边缘

修复

1. 提高码率（唯一根治）

2. 限制QP：MaxQP=32

3. 8×8变换：8x8dct=1

4. 预处理模糊（牺牲锐度）

代码

param->analyse.b_transform_8x8=1;

成因分析

4:2:0色度分辨率=亮度1/4。鲜艳色块边缘，色度通道不够精细 → 颜色"溢出"。

修复

1. 切换4:2:2（码率+50%）

2. chroma_qp_offset=-2

代码

void ordered_dither(
  uint8_t *y, int w, int h) {
  static const int b[4][4] = {
    {0,8,2,10},{12,4,14,6},
    {3,11,1,9},{15,7,5,13}
  };
  for(int i=0;i<h;i++)
    for(int j=0;j<w;j++){
      int v=y[i*w+j]
        +b[i%4][j%4]*16/256-8;
      y[i*w+j]=CLAMP(v,0,255);
    }
}

成因

• 双缓冲/三缓冲
• DRM/KMS page flip
