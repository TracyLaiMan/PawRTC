第一篇：图像采集

从光子到数字信号——传感器的物理世界

P1CMOS 传感器光电转换
光子穿过镜头 → 微透镜(增加聚光) → 彩色滤光片(Bayer CFA) → 硅光电二极管 → 光电转换产生电子 → 源极跟随器转电压 → ADC 量化为数字值。整个链条决定了传感器的极限画质。
光子 → 电子(e⁻) → 电压(V) → 数字(DN)

像素结构

每个像素包含：
• 光电二极管：光子→电子，效率由量子效率QE决定
• 传输门TG：控制电子到FD节点
• 浮置扩散FD：电荷→电压转换
• 源极跟随器SF：电压缓冲
• 选择门RS：行选通
• 复位门RST：清空FD电荷

// 信号链
光子 → QE×光子数=电子数
电子 → V_sig = Q_e/C_FD
V_sig → ADC → RAW10/12

关键参数

量子效率 QE：每个光子产生电子的概率。绿光~70%，红光~50%，蓝光~40%

满阱容量 FWC：像素能存储的最大电子数。大像素=高FWC=高动态范围。IMX307: ~15ke⁻, IMX335: ~20ke⁻

读出噪声：读出电路引入的电子数。低读出噪声=暗光画质好。IMX307: ~2e⁻

SNR_max = 20×log10(√FWC)
IMX307: 20×log10(√15000)
≈ 41.8 dB

全局快门 vs 卷帘快门

全局快门(Global)：所有像素同时曝光→无果冻效应→但需要每个像素一个采样保持电容→填充因子降低→灵敏度下降30%→成本高3-5倍

卷帘快门(Rolling)：逐行曝光→果冻效应→但像素面积全部用于光电转换→灵敏度高→成本低→监控摄像头几乎都用卷帘

暗电流与热噪声

暗电流：无光时硅中热生载流子，温度每升高8°C翻倍。
影响：长曝光时暗电流积累→像素饱和（热像素）
对策：暗帧减法（关闭快门拍一帧做减法）或BLC自动扣除

暗电流(T) = I₀ × 2^((T-T₀)/8)
// T₀=25°C, I₀=10 pA/cm² 典型值
曝光20ms 增益5x
P2传感器控制与数据接口
SoC 通过 I2C/SPI 配置传感器寄存器（曝光/增益/模式），传感器通过 MIPI CSI-2 将 Bayer RAW 数据高速传输到 ISP。两者分离设计：低速控制链 + 高速数据链。
I2C: 寄存器R/W → MIPI CSI-2: RAW10/Bayer 数据流

I2C 控制接口

• SCL/SDA：标准 I2C，速率通常 100K/400K
• 传感器地址：通常 0x1A 或 0x34
• 关键寄存器组：
  曝光时间 (0x0202)
  模拟增益 AGC (0x0204)
  数字增益 DG (0x020e)
  帧率/模式 (0x0301)
  IR-Cut 触发 (GPIO)
• 写入延迟：1帧后才生效

IMP_ISP_Tuning_SetAeAttr()
IMP_ISP_Tuning_SetWbAttr()

MIPI CSI-2 数据接口

D-PHY：差分信号，每 Lane 1.5-2.5Gbps
• 2-Lane: ~4Gbps → 1080p30 RAW10 需 ~1.5Gbps
• 4-Lane: 8Gbps → 4K30

C-PHY：3线一组，每 Trio ~2.5Gbps
• 更省 Lane 数，但实现复杂

数据包：PH(包头) + Payload + PF(包尾)
• LP(低功耗) → HS(高速)切换需时间

BW = width × height × fps × bpc × lanes
1080p30 RAW10 2lane:
= 1920×1080×30×10/2
= 311 Mbps/lane ✓

时序与同步

• VSYNC/HSYNC/PCLK：并行接口已少用
• MIPI LP/HS：帧起始(FS)/帧结束(FE)包标记边界
• Embedded Data：部分传感器在影像数据前追加寄存器信息帧
• WDR模式：DOL交替输出长/短曝光行

常见问题

• Lane 数配置错误 → 画面花屏/撕裂
• 时钟频率不匹配 → 无法锁定
• I2C 写入后未等1帧 → 参数不生效
• MIPI LP→HS 切换时间过长 → 丢行
• 传感器断电顺序错误 → 残影

IMX307 切换 WDR 模式需要先 stop stream 再配置，否则寄存器写入无效。
P3Bayer 格式与 RAW 数据
传感器输出的每个像素只有 R/G/B 中的一个通道，按 Bayer 排列。这是后续所有 ISP 处理的起点数据——理解 Bayer 格式是理解整个 ISP 的基础。
输出: RAW8/10/12 Bayer (RGGB/BGGR/GRBG/GBRG)

Bayer 排列格式

4种标准排列，区别在于 (0,0) 位置的通道：
• RGGB：最常见，Sony IMX 系列多用
• BGGR：IMX307 使用此格式
• GRBG / GBRG：较少见

规律：每行必是 R/G 或 G/B 交替，G 的数量是 R/B 的 2 倍——因为人眼对绿色最敏感。

// BGGR 示例 (IMX307)
B G B G B G
G R G R G R
B G B G B G
G R G R G R

量子效率与白平衡

硅的 QE 曲线决定了传感器对光的响应：
• 绿光 QE 最高 (~70%)
• 红光次之 (~50%)
• 蓝光最低 (~40%)

这意味着：同一个白色物体，R/G/B 通道的原始输出不同 → 必须 AWB 增益校正。蓝色需要最大增益 → 蓝通道噪声最大。

R_gain = G_avg / R_avg ≈ 1.4
B_gain = G_avg / B_avg ≈ 1.75
// 蓝通道增益最大 → 噪声放大最多

RAW 位深选择

RAW8：最快，带宽最小，但暗部量化台阶大
RAW10：主流选择，噪声基底内包16个量化级
RAW12：HDR场景，保留更多暗部细节

经验：如果传感器读出噪声>4 DN，RAW10已足够。

压缩与传输

MIPI CSI-2 支持 RAW 压缩：
• RAW10 compact：4个10bit像素打包5字节
• 节省 20% 带宽 vs 不压缩
• 10bit有效位不变

部分平台支持 DDR 直接写入 RAW10 compact 格式。

T31 的 MIPI RX 支持 1/2/4 lane，RAW10 compact 模式下 2-lane 可支持 1080p60。

第二篇：ISP 处理链路

从 Bayer RAW 到 YUV——10 级硬件流水线

01黑电平校正 BLC
减去传感器固定偏移，恢复零光基准。不做则全链偏移。
RAW10 Bayer → 黑电平偏移64 → RAW10 Bayer (扣除后)

算法

pixel_out = pixel_in - BLC_offset
// OB区实时统计: BLC_offset = mean(OB_pixels)
// 10bit 典型值: 64

踩坑

• 偏大 → 整体偏暗
• 偏小 → 暗区发灰
• Gr/Gb不一致 → Green Imbalance

IMX307 Day/Night切换需重标BLC
02镜头阴影校正 LSC
校正渐晕(cos⁴θ衰减)和边缘偏色(色散)，通过增益表逐像素补偿。
RAW10 Bayer → × 通道增益表 → RAW10 Bayer

增益表法

pixel_out = pixel_in × Gain[r][c][ch]
// 17×17 或 33×33 网格
// 多光温标定: D50/A/TL84

踩坑

• Table不准 → 四角偏色
• 广角镜头增益>3x → 边缘噪声飙3倍
• 切光温未切换Table

03坏点校正 BPC
检测并替换静态/动态坏点，防止Demosaic产生十字伪影。
RAW10 Bayer → 检测+中值替换 → RAW10 Bayer

检测

grad = |P(x,y) - median(same_ch_neighbors)|
if grad > Th(ISO): mark bad
Th 随增益动态调整

关键排序

BPC 必须在 Demosaic 之前，否则坏点扩散为 5×5 十字伪影。

60°C+ 动态坏点爆发
043A 统计 AE/AWB/AF
硬件实时统计亮度直方图和分区RGB均值，唯一有闭环反馈的处理级。
RAW10 Bayer → 统计直方图+分区均值 → 反馈控制传感器

AE 控制

error = Y_target - Y_avg
new_exp = cur_exp × (1+Kp×error)
可用增益 = 曝光 + AGC + DG
约束: AWB的DG_ratio < 4x

AWB 控制

R_gain = G_R / R_avg
B_gain = G_B / B_avg
// 灰世界/完美反射/查找表
05去马赛克 Demosaic
传感器每个像素只捕获 R/G/B 中的一种颜色（Bayer CFA），Demosaic 通过邻域插值恢复出每个像素的完整 RGB 三通道值。这是 ISP 中计算量最大、对画质影响最关键的环节。
RAW10 Bayer (1ch/px) → 方向梯度插值 → RGB888 (3ch/px)

双线性插值（最简单）

以 G 位置恢复 R 为例：

R_at_G = (R_left + R_right) / 2 // 水平
// 或 (R_up + R_down) / 2 垂直
// 或 4邻域均值（对角位置）

优点：计算量小。缺点：边缘处产生拉链伪影（zipper artifact）和伪色（false color）。

边缘感知插值（实际使用）

沿边缘方向插值，避免跨边缘混合：

grad_H = |G_left - G_right|
grad_V = |G_up - G_down|
if grad_H < grad_V:
R_at_G = (R_left + R_right) / 2 // 水平
else:
R_at_G = (R_up + R_down) / 2 // 垂直

避免了跨边缘插值产生的拉链效应。

高级算法

• Hamilton-Adams：梯度+色差恒定假设
• ALSC（Adaptive）：自适应方向选择
• 频域方法：利用 Bayer 采样特性在频域分离
• 板端多用硬件实现的方向梯度法

Demosaic 质量直接决定锐度和伪色水平。低端 ISP 用双线性 → 远处栅栏/树叶大量摩尔纹。

与上下游的衔接

• 前置依赖：BPC 必须在 Demosaic 之前完成→否则坏点扩散为 5×5 十字伪影
• 输出格式：从此级开始数据从 RAW 变为 RGB，后续所有处理都在 RGB 域
• 硬件实现：通常需要 5×5 邻域窗口，多行 Line Buffer

Bayer RAW (1ch/px)Demosaic RGB (3ch/px)
06颜色校正矩阵 CCM
传感器的 RGB 光谱响应与人类视觉的 XYZ 基色不匹配 → 拍出的颜色不准。CCM 通过 3×3 矩阵将传感 RGB 空间映射到标准 sRGB 空间，是颜色准确的数学核心。
RGB888 → × 3×3矩阵 → RGB888 (sRGB色彩空间)

数学原理

[R'] [c00 c01 c02] [R] [offset_R]
[G'] = [c10 c11 c12] [G] + [offset_G]
[B'] [c20 c21 c22] [B] [offset_B]

9 个系数 + 3 个偏移，共 12 个参数。不同色温下矩阵不同。

标定方法

1. 拍摄 ColorChecker 标准色卡
2. 提取 24 个色块的实际 RGB
3. 最小二乘拟合: C = RGB_sensor⁻¹ × RGB_target
4. 通常需多组（D50/A/TL84）
5. AWB 运行时根据色温插值选取 CCM

饱和度控制

C_sat = α × C_identity + (1-α) × C_calibrated
// α=0: 完全校正
// α=1: 无校正(恒等矩阵)
// α∈[0.1, 0.3]: 自然肤色

常见问题

• CCM 不匹配 → 肤色偏绿/偏红
• 高饱和区域溢出 → 需 clamp 或软饱和
• 色温切换时矩阵跳变 → 需平滑插值

安防场景优先保证绿色（草地/树木）和肤色准确，可牺牲部分高饱和度还原。
传感器原始色CCM 校正后(sRGB)
07伽马校正 Gamma
相机传感器线性响应光线，但人眼对暗部变化更敏感（对数响应）。Gamma 校正将线性信号映射为非线性，充分利用有限的位深，让暗部有更多量化层级。
RGB888 (线性) → LUT查表 → RGB888 (非线性)

数学原理

V_out = V_in ^ γ
// 典型 γ = 0.45 (= 1/2.2)
// 编码 Gamma < 1：提亮暗部
// 显示 Gamma ≈ 2.2：还原亮度
// 端到端: 0.45 × 2.2 ≈ 1.0

实际 ISP 使用 LUT（查找表），支持自定义曲线。

为什么要非线性编码？

8bit 线性编码：暗部仅 10-20 个层级 → 量化梯度大 → 色带明显。

8bit Gamma 编码：暗部有 100+ 层级 → 平滑过渡 → 人眼感知质量好。

本质上是一种感知域的压缩——把有限的 bit 预算用在人眼最敏感的暗部。

自定义 Gamma 曲线（安防）

安防场景常调整 Gamma 曲线：
• 黑电平拉伸 → 暗部更清楚
• 高光压缩 → 窗外不过曝
• S 型曲线 → 增加对比度

// 典型安防 Gamma
暗部: γ > 0.45 (提更多)
中间: γ ≈ 0.45
亮部: γ < 0.45 (压一点)

与 WDR 的关系

WDR 场景下 Gamma 曲线设计更关键：
• 需同时映射 100dB+ 动态范围到 8bit
• 全局 Gamma 不够 → 需局部 Tone Mapping
• Gamma + CLAHE 组合效果更好

08降噪 + 锐化 NR / EE
NR 和 EE 是一对矛盾体：降噪模糊细节，锐化放大噪声。ISP 的核心挑战就是在两者间找到最优平衡。这一级通常是可调参数最多、对主观画质影响最大的环节。
RGB888 → 双边/时域滤波 + Unsharp Mask → RGB888

2D 空域降噪

// 双边滤波（保边缘）
w_i = exp(-|P_i - P_c|²/(2σ_r²))
× exp(-|x_i - x_c|²/(2σ_s²))
// σ_r: 亮度差阈值 → 控制保边程度
// σ_s: 空间范围 → 控制平滑范围

3D 时域降噪

// 运动自适应时域融合
if |P_cur - P_prev| > T_motion:
α = 1.0 // 运动区域不融合
else:
α = σ_noise / max(|diff|, σ_noise)
P_out = α × P_cur + (1-α) × P_prev

运动区域退化回2D降噪 → 噪声更粗糙

边缘增强（锐化）

// Unsharp Mask
HP = P_original - P_lowpass
P_out = P_original + gain × HP

// 自适应增益
gain = f(luma, edge_strength)
// 暗区增益低（避免放大噪声）
// 边缘区域增益高
// 平坦区域增益低（避免蚊子噪声）

NR + EE 联合调优

• 先 NR 后 EE → 锐化会放大残余噪声
• 先 EE 后 NR → 降噪模糊边缘
• 实际：NR-EE 级联 + 自适应切换
→ 平坦区重降噪轻锐化
→ 边缘区轻降噪重锐化

最常见的画质投诉：远景"油画感"= 过度降噪 + 过度锐化 → 细节消失但边缘明显。
含噪声NR + 轻锐化
NR强度5 锐化3
09色彩空间转换 CSC
ISP 内部处理在 RGB 域完成，但视频编码器（H.264/H.265）工作在 YUV 域。CSC 模块完成 RGB → YCbCr 转换，并可选下采样色度（4:4:4 → 4:2:0），为编码器准备数据。
RGB888 → YCbCr (BT.601/709) → 下采样 → YCbCr 4:2:0

BT.601 转换公式（SD）

Y = 0.299R + 0.587G + 0.114B
Cb = -0.169R - 0.331G + 0.500B + 128
Cr = 0.500R - 0.419G - 0.081B + 128

BT.709 用于 HD（1080p+），系数略有不同。

色度下采样

4:2:0 模式下，Cb/Cr 在水平和垂直方向各取 1/2：

Cb_420[x/2][y/2] = avg(Cb_444[x][y], Cb_444[x+1][y],
Cb_444[x][y+1], Cb_444[x+1][y+1])

这是色度拖尾的根源——鲜艳颜色的边缘分辨率降低 4 倍。

10宽动态 / 色调映射 WDR / TM
真实场景动态范围 120dB+，传感器仅 60-70dB。WDR 通过多曝光融合扩展动态范围，Tone Mapping 将高动态信号压缩到显示范围。这是安防 IPC 最核心的差异化能力。
多帧 RAW/YCbCr422 HDR → TM压缩 → YCbCr420 SDR

多曝光融合

pixel_out = w_short × pixel_short + w_long × pixel_long
// 权重 w 随像素亮度自适应：
// 暗区 → w_long 占优
// 亮区 → w_short 占优
// 运动区域 → 降低融合权重防鬼影

DOL (Digital Overlap)

单帧交替读出长短曝光行：
• 无时差 → 无鬼影
• 垂直分辨率减半
• IMX335 支持 2-exp DOL
• 比"帧交替融合"更先进

Tone Mapping

// 全局 TM
V_out = V_in ^ (1/γ_global)

// 局部 TM（更好）
Base = Lowpass(V_in) // 基底
Detail = V_in / Base // 细节
Base_out = Base ^ (1/γ_local)
V_out = Base_out × Detail

常见问题

• 融合鬼影 → 运动物体产生重影
• 色调映射过度 → 画面发灰
• 暗区提亮后噪声放大 → 需配合 NR
• 色晕（Halo）→ 局部TM过度增强边缘

安防场景 WDR 核心指标：逆光人脸可辨识 + 窗外车牌可读。
单曝光（暗区全黑）WDR 融合 + TM

第三篇：H.264 / H.265 视频编码

从 YUV 帧到压缩码流——预测、变换、量化、熵编码的完整闭环

E1编码器架构总览
视频编码的核心思想：预测 + 残差编码。先预测当前块（利用帧内/帧间相关性），只编码预测残差（远小于原始值），再通过变换量化进一步压缩，最后熵编码消除统计冗余。

关键公式：码率 = 空间相关性（帧内） + 时间相关性（帧间） + 统计冗余（熵编码） + 感知冗余（量化）→ 四重压缩。

编码器处理环路

1. 预测：帧内(空间)/帧间(时间)→得残差
2. 变换：DCT将残差转到频域
3. 量化：丢弃高频分量（损失来源）
4. 熵编码：CAVLC/CABAC无损压缩
5. 环路滤波：重建帧去块效应
6. 重建：反量化+反变换+预测→参考帧

H.264 vs H.265 关键差异

• 宏块: 16×16 → CTU: 64×64
• 帧内模式: 9种 → 35种
• 运动Merge: 无 → Merge/Skip
• 变换: 4×4 → 4×4/8×8/16×16/32×32
• 环路滤波: Deblocking → Deblocking+SAO
• 编码效率: H.265 比 H.264 节省 ~40% 码率
• 计算量: H.265 编码慢 3-5 倍

E2帧类型与 GOP 结构
I帧(帧内)完全自包含，P帧(前向预测)参考前一帧，B帧(双向)参考前后帧。GOP的I帧间隔决定随机访问粒度和错误恢复速度。

帧类型详解

I帧(Intra)：仅帧内预测，码率最大，可随机访问
IDR帧：特殊I帧，强制清空DPB→其后P帧不引用IDR前帧
P帧(Predicted)：参考前一帧，码率约为I帧1/3
B帧(Bi-predicted)：参考前后两帧，码率最小，但引入延迟
Skip/Direct：P/B帧中直接复制参考块，码率极低

GOP 结构与权衡

封闭GOP：IDR→P→P→…→IDR
• 优点：IDR后可独立解码
• 缺点：IDR处码率突增

开放GOP：I帧(非IDR)→B→P→B→P→…
• 优点：码率平滑
• 缺点：随机访问需要前一个IDR

安防经验：IDR间隔15-30帧，B帧不用（延迟+算力）

帧类型与参考关系
E3帧内预测
利用图像空间相关性：相邻像素值相近。用已编码块的边界像素预测当前块，只编残差。H.264提供9种4×4+4种16×16模式，H.265扩展到35种。

H.264 帧内模式

4×4 亮度块：9种方向
• 模式0: 垂直
• 模式1: 水平
• 模式2: DC
• 模式3-8: 对角线方向

16×16 亮度块：4种
• 垂直/水平/DC/平面

8×8 色度块：4种（同16×16）

// 预测值 = 已编码边界像素加权
Mode0(V): pred[x][y] = left_top[x]
Mode1(H): pred[x][y] = left_col[y]

H.265 帧内增强

35种预测模式：
• 模式0: Planar（加权平均）
• 模式1: DC
• 模式2-34: 33种角度方向
• 角度间隔约6.7°

RDO 模式选择：
代价 = 码率 + λ × 失真
遍历所有模式选代价最小的
• 帧内预测占了 H.265 编码 60%+ 算力

板端编码器通常简化RDO：只检查5-8种候选模式
E4帧间预测与运动估计
视频相邻帧高度相似→用前一帧预测当前帧，只编码运动矢量+残差。运动估计是编码器计算量最大的模块（占40%+算力），决定了压缩效率上限。

运动估计流程

1. 整像素搜索：在参考帧搜索窗口找最佳匹配
• 钻石/六边形搜索（非全搜索）
• SAD = Σ|cur - ref|（绝对差之和）

2. 亚像素搜索：1/2像素插值→1/4像素
• 6抽头FIR滤波器(1/2像素)
• 双线性插值(1/4像素)

3. 运动矢量编码：MVD = MV - MVP(预测MV)

bestMV = argmin SAD(cur, ref(x+dx,y+dy))
cost = SAD + λ×bits(MVD)

高级帧间工具

Skip 模式：直接复制参考块，无残差，码率极低
适合：静止背景/慢速运动

H.265 Merge 模式：从候选列表选MV，不编MVD
• 候选来源：空域/时域相邻块
• 大幅减少运动矢量编码开销

B帧双向预测：前向+后向加权平均
• 权重可自适应（WP）
• 低延迟场景不用B帧

安防板端不用B帧：2帧延迟+2×运动估计算力
E5变换与量化
DCT变换将预测残差的能量集中到低频系数，量化丢弃感知不敏感的高频分量。这是视频编码唯一的失真来源，也是码率控制的核心旋钮。

整数 DCT 变换

H.264：4×4 整数DCT
• 无乘法，只有加法和移位
• 核心思想与传统DCT一致但避免浮点

H.265：4×4 / 8×8 / 16×16 / 32×32
• 大块DCP更好的能量集中
• 4×4 亮度残差可用DST替代DCT

// 4×4 整数DCT核心
Y = C×X×Cᵀ
// C为整数矩阵，X为残差
// 变换后低频集中在[0][0]

量化 — 失真的唯一来源

量化步长与QP关系：

QStep = 0.625 × 2^(QP/6)
QP ∈ [0, 51]
QP每+6 → QStep翻倍
逆量化: coeff_round(QP / QStep)

码率-失真关系：
• QP↓ → 码率↑ 画质↑
• QP↑ → 码率↓ 画质↓
• QP 28以下接近无损
• QP 36+ 块效应可见
• QP 42+ 严重马赛克

板端建议：MaxQP=36, MinQP=10

E6熵编码
无损压缩的最后一步：利用符号的统计特性进一步压缩。H.264支持CAVLC和CABAC，H.265只支持CABAC。CABAC比CAVLC节省5-15%码率，但计算量更大。

CAVLC (H.264 Baseline)

上下文自适应变长编码：
1. Z形扫描4×4系数块
2. 统计非零系数个数/尾随1个数
3. 查表选择VLC码表
4. 编码每个系数的level和run

// 扫描顺序（Zigzag）
0 1 5 6
2 4 7 12
3 8 11 13
9 10 14 15

CABAC (H.264 Main / H.265)

上下文自适应二进制算术编码：

1. 二值化：将语法元素转为0/1串
  • 一元码 / 截断一元 / 指数Golomb
2. 上下文建模：根据邻块状态选择概率模型
  • 亮度/色度/残差分类
3. 算术编码：按概率区间编码bit
  • 旁路模式：0.5概率均匀编码

CABAC是H.265比H.264省码率的关键模块之一
E7码率控制与 NAL 输出
码率控制决定每一帧/宏块的QP值，在目标码率和画质间平衡。输出封装将编码数据打包为NAL单元，按 Annex B 或 AVCC 格式输出。

码率控制模式

CBR：固定码率，每帧码率恒定
• 适合带宽确定场景
• VBV缓冲水位控制关键

VBR：可变码率，画质恒定
• 复杂帧多分配，简单帧少分配
• 峰值码率可能超标

CRF：恒定质量(x264独有)
• QP = CRF + 内容复杂度偏移
• 安防常用 CRF 23-28

// VBV缓冲约束
bits_per_frame ≤ VBV_size / delay
VBV_fullness += bits_input
VBV_fullness -= bits_frame
if VBV_fullness > VBV_max: 溢出
if VBV_fullness < 0: 下溢

NAL 单元封装

H.264 NAL 结构：
• Start Code: 00 00 00 01 或 00 00 01
• NAL Header: 1字节
  forbidden_zero_bit(1) + nal_ref_idc(2) + nal_unit_type(5)
• RBSP: 原始字节序列载荷

关键NAL类型：
• 7: SPS (序列参数集)
• 8: PPS (图像参数集)
• 5: IDR帧
• 1: 非IDR帧
• 6: SEI补充信息

Annex B vs AVCC：
• Annex B: Start Code前缀（流式）
• AVCC: 长度前缀（容器封装）
• SRTP/RTP传输用 Annex B

LiveKit推流必须每个IDR前发送SPS/PPS
