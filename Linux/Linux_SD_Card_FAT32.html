2. SD 卡硬件接口 — 引脚 · 电气特性 · PCB 布局

SD 卡不只是软件协议——硬件层面的信号质量、供电和 PCB 布局直接决定了卡能否稳定工作在高速模式。软件调不通时，70% 是硬件问题。

引脚定义 — 9-pin 物理接口

SD 模式 vs SPI 模式 — 同一组引脚，不同功能

SD 模式 (4-bit)：CLK (Pin5, Host→Card 时钟)、CMD (Pin2, 双向命令/响应)、DAT0 (Pin7, 双向数据)、DAT1 (Pin8)、DAT2 (Pin9)、DAT3 (Pin1)。
SPI 模式：CLK→SCLK, CMD→MOSI, DAT0→MISO, DAT3→CS。SPI 模式仅 1-bit 且无 CRC 保护——速率低但 MCU 可直接驱动（无需 SD 控制器）。
其他引脚：Pin4=VDD (3.3V 供电), Pin3/6=GND, Pin10-11=卡检测 (机械开关)。DAT1 在 SDIO 模式复用为中断线。DAT3 在初始化前用作卡检测 (50kΩ 上拉)。

电压域与供电要求

3.3V 默认 · 1.8V 高速 · 双电源域 · 去耦

VDD (Pin4)：主供电 2.7V~3.6V。初始化阶段固定 3.3V。UHS-I 模式下信号切换到 1.8V 但 VDD 保持 3.3V——信号电压和供电电压是两回事。
VDDIO (Host 侧)：SoC 的 SD 控制器 I/O 电源域——必须可切换到 1.8V (通过 GPIO 控制的 LDO 或 PMIC 输出)。内核通过 vqmmc-supply 描述此电源。
去耦电容：VDD 引脚附近必须放置 0.1μF + 10μF 并联——卡座的 VDD 到 GND 之间走线尽可能短。SD 卡在写入时电流尖峰可达 200mA/100ns——无去耦→电压跌落→CRC 错误。

信号电平与上拉电阻

开漏(OD) vs 推挽(PP) · 上拉强度 · 电平转换

CMD 和 DAT 线有双重驱动模式：初始化阶段 (≤400kHz) 使用 开漏模式——Host 和 Card 都可以拉低总线。需要外部上拉电阻 (10kΩ~100kΩ) 拉到 VDDIO。数据传输阶段切换到 推挽模式——驱动能力强、边沿陡、适合高速。
上拉电阻值选择：阻值太大 (100kΩ)→上升沿太慢 (RC 时间常数太大→边沿不满足时序)；阻值太小 (1kΩ)→静态功耗大。推荐 10kΩ~50kΩ。
SoC 内部上拉：多数 SoC 的 SD 控制器内置可配置上拉 (如 Rockchip 的 bias-pull-up)。设备树中 bias-pull-up 属性启用。

时钟树 — SD 卡时钟从哪来

SoC PLL → SD 控制器分频器 → CLK 引脚

SD 卡的 CLK 由 SoC 的 SD/MMC 控制器内部时钟分频器生成——时钟源通常来自系统 PLL (如 400MHz) → 经分频器产生目标频率。初始化时要求 ≤400kHz——通常分频到 375kHz (400MHz/1066)。高速模式需要 50MHz (400MHz/8)，SDR104 需要 208MHz——PLL 需支持此频率。
关键约束：① 时钟占空比必须是 50%±5%（SD 规范要求）② 时钟必须在 CMD0 之前就开始 toggle（≥74 个周期）③ 频率切换必须平滑——不能有毛刺（glitch-free clock mux）。设备树中通过 max-frequency 和 assigned-clock-rates 配置。

PCB 布局要点

等长走线 · 阻抗控制 · 串扰隔离 · 参考层

走线长度匹配：CLK 和 CMD/DAT0~3 的走线长度差应 ≤ 10mm (50MHz 时，10mm → ~50ps 延迟差 → 远小于半个时钟周期 10ns——安全裕度充足)。SDR104 (208MHz) 时要求更严 ≤ 5mm。
特性阻抗：SD 信号线设计为 50Ω ±10% 单端阻抗。串联端接电阻：在 Host 侧每条信号线上串联 22Ω~33Ω——抑制信号反射和过冲。放置位置靠近 Host (SoC) 而非卡座。
串扰隔离：CLK 是敏感信号——与 DAT 线间距 ≥ 3× 线宽。CLK 两侧用地线包围（guard trace）。如果 CLK 耦合到 DAT——数据采样时刻的电平跳变→误码。
地参考平面：SD 信号下方必须有连续的地平面——不能跨越地平面分割。分割地→回流路径断裂→EMI 辐射↑ + 信号完整性↓。

⚠ SD 卡硬件调试 Checklist — 示波器逐项排查
1. 供电：VDD 纹波 < 50mVpp (用示波器 AC 耦合测)。写入时如果纹波 >100mV → 去耦不足。
2. CLK 信号：示波器探头点 CLK 引脚——确认电压摆幅 (≥ 0.75×VDDIO)、占空比 (45%~55%)、无振铃/过冲。400kHz 初始化时 CLK 应有稳定方波——如果没波形→检查 SoC 时钟配置。
3. CMD 响应：发 CMD0 后 CMD 线上应有响应脉冲 (卡在 Idle 状态回复 0x01)。如果 CMD 一直高→卡未上电或未复位→查 VDD 和 CMD 上拉。
4. DAT 线：未选中卡时 DAT0-3 应该被上拉电阻拉到高电平。如果某条 DAT 线一直低→可能是焊接短路或 IO 方向配置错误。
5. 卡检测：cat /sys/kernel/debug/mmc0/ios 查看 card detect 状态。如果用 GPIO 检测→cat /sys/class/gpio/gpioXX/value 确认插拔时电平变化。

3. SD 卡协议 — 初始化序列与命令体系

上电 → CMD0 → CMD8 → ACMD41 → CMD2 → CMD3 → SCR → 总线切换 — 从 Idle 到 Transfer 状态

上电与识别阶段 (≤400 kHz)

Card Identification Mode

① 上电 VDD → 等待 250ms 稳定
② 发送 ≥74 个时钟周期 (CMD 线保持高)
③ CMD0 (GO_IDLE_STATE) → 所有卡复位
④ CMD8 (SEND_IF_COND): 参数=VHS+0xAA，检查电压+SD 版本 (v2.00+)
⑤ ACMD41 (SD_SEND_OP_COND): HCS=1 (支持 SDHC/XC)，S18R=1 (请求 1.8V)，轮询直到 bit31=1 表示就绪
⑥ (可选) CMD11 VOLTAGE_SWITCH → 切 1.8V 信令

数据传输阶段 (升至高速)

Data Transfer Mode

⑦ CMD2 (ALL_SEND_CID) → 读 128-bit 卡标识
⑧ CMD3 (SEND_RELATIVE_ADDR) → 分配 RCA，进入 Stand-by
⑨ CMD9 (SEND_CSD) → 读 CSD 获取容量/速度/版本
⑩ ACMD51 (SEND_SCR) → 读 SD 配置寄存器 (支持的总线宽度)
⑪ ACMD6 (SET_BUS_WIDTH) → 切 4-bit
⑫ CMD6 (SWITCH_FUNC) → 切 High Speed / UHS 模式
⑬ CMD7 (SELECT_CARD) → 选中卡，进入 Transfer

Idle (CMD0 后) → Ready (ACMD41 完成后) → Ident (CMD2 后) → Stby (CMD3 后) → Transfer (CMD7 选中后) → Data (CMD17/18 读或 CMD24/25 写时) → 完成后回到 Transfer

5. FAT32 磁盘布局 — BPB · FAT 表 · 目录项 · 簇链

从第 0 扇区到文件数据 — FAT32 的五层磁盘结构完整解析

BPB — BIOS 参数块

扇区 0 — 卷的元数据

Bytes Per Sector (偏移 0x0B): 通常 512
Sectors Per Cluster (0x0D): 2^n (1/2/4/8...128)
Reserved Sectors (0x0E): FAT32 通常 32
Number of FATs (0x10): 通常 2 (FAT1 + FAT2 互为备份)
Sectors Per FAT (0x24): FAT32 用此字段
Root Cluster (0x2C): 根目录起始簇号 (通常 2)
FS Info Sector (0x30): 通常 1 (含空闲簇计数)

FAT 表 — 簇分配链

簇 → 簇的单链表

FAT32 每个表项 32-bit (高 4 位保留 = 28-bit 有效)。
0x00000000: 空闲簇
0x00000002–0x0FFFFFEF: 下一个簇号
0x0FFFFFF8–0x0FFFFFFF: EOF (文件结束)
0x0FFFFFF7: 坏簇
例: 文件占簇 3→5→7→0x0FFFFFFF 表示 FAT[3]=5, FAT[5]=7, FAT[7]=EOF

目录项 — 32 字节固定

文件名 + 属性 + 首簇号

偏移 0x00: 短文件名 (8.3, 11 字节)
0x0B: 属性 (0x10=目录, 0x20=存档, 0x0F=LFN)
0x14: 首簇号高 16 位 (FAT32)
0x1A: 首簇号低 16 位
0x1C: 文件大小 (字节, 32-bit)
VFAT LFN: 长文件名通过多个 32 字节槽位存储 (每槽 13 个 UTF-16LE 字符)，属性=0x0F

6. Linux VFAT 驱动 — fs/fat/ 源码架构

内核中 FAT 文件系统的实现 · 超级块/Inode/FAT 表操作 · 簇分配与回收

源码文件

fs/fat/ 目录结构

inode.c — inode 读写 (fat_get_block 映射簇→扇区)
cache.c — FAT 表缓存 (以扇区为单位)
fatent.c — FAT 表项操作 (fat_ent_read/write/blocknr)
misc.c — 簇管理 (fat_alloc_clusters/fat_free_clusters)
dir.c — 目录遍历 (fat_search_long/fat_add_entries)
namei_vfat.c — VFAT 文件名处理 (8.3 + LFN 编解码)
file.c — 文件读写

关键数据结构

msdos_sb_info — 超级块私有数据

fatent_operations: FAT12/16/32 的统一访问接口。三个实现: fat12_ops, fat16_ops, fat32_ops。
msdos_sb_info: 挂载时从 BPB 填充 — fat_bits, cluster_size, root_cluster, prev_free, free_clusters。
FAT 缓存: 以 FAT 扇区为单位缓存，LRU 回收，避免每次都读磁盘。

关键操作

簇分配 · 释放 · 读写

分配 (fat_alloc_clusters): 从 FAT 表扫描空闲簇 → 建立簇链 → 更新 FAT 表 → 标记脏
释放 (fat_free_clusters): 遍历簇链 → FAT[cluster]=0 → 更新 FS Info 空闲计数
读 (fat_get_block): 文件偏移 → 逻辑簇号 → FAT 链遍历 → 物理扇区号 → map_bh 返回

⚠️ FAT32 的限制
最大文件大小: 4 GB − 1 字节 (32-bit 文件大小字段的硬限制) — 无法存储超过 4GB 的单个文件。
最大卷大小: 2 TB (理论), Windows 格式化工具限制为 32 GB。
无 Unix 权限: FAT 不支持 uid/gid/mode — 挂载时通过 mount 选项统一设定。
无日志: 突然断电可能导致 FAT 表损坏 — fsck.vfat 可修复但可能丢数据。
碎片化: 长期使用后文件碎片增加，需要定期检查或在 Windows 下整理。
