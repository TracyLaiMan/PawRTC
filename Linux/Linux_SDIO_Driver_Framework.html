SDIO 协议

SD Association 规范

Secure Digital I/O 扩展 SD 物理总线，使同一接口不仅能访问存储卡，还能连接 WiFi、蓝牙、GPS、UART 等 I/O 功能设备。一张 SDIO 卡最多支持 7 个 I/O 功能。

与 MMC 子系统的关系

drivers/mmc/core/sdio*.c

SDIO 是 MMC 子系统的三大协议之一（MMC / SD / SDIO）。Core 层解析 CCCR / FBR / CIS 寄存器，创建 sdio_func 设备。函数驱动通过 sdio_bus 匹配并驱动具体 I/O 功能。

典型设备

WiFi · BT · GPS · UART

Broadcom FullMAC (brcmfmac)、Marvell (mwifiex)、Qualcomm Atheros (ath6kl)、TI WiLink — 这些都是经典的 SDIO 接口 WiFi/BT 芯片。Android/iOS 设备广泛使用 SDIO WiFi。

1. SDIO 协议基础

SDIO 卡类型 · CIA 寄存器体系 · CCCR · FBR · CIS — 理解 SDIO 必须先掌握这三个寄存器结构

SDIO 卡类型

四种卡分类

全速 (Full-Speed): 1-bit/4-bit，≤25MHz (SD) 或更高
低速 (Low-Speed, LSC=1): 仅 1-bit (除非 4BLS)，≤400kHz
纯 I/O 卡: 无 Memory 部分，仅响应 CMD5
Combo 卡: 同时含 SD Memory + SDIO I/O

CIA — 通用 I/O 区域

Function 0 的 128KB 地址空间

CCCR (0x000–0x0FF)：卡通用控制寄存器
FBR (0x100–0x7FF)：每 Function 256 字节基础寄存器
CIS (0x1000–0x17FFF)：卡信息结构（Tuple 链表）
主机通过 CMD52(单字节) 和 CMD53(多字节) 读写 CIA。

标准功能接口码

FBR 0x00 字段

0x01 UART · 0x02 Bluetooth Type-A
0x03 Bluetooth Type-B · 0x04 GPS
0x05 Camera · 0x06 PHS
0x07 WLAN (WiFi) · 0x08 SDIO-ATA
0x0F 扩展 (见 FBR 0x01)

/* ===== CIA (Common I/O Area) 完整内存映射 — Function 0 ===== */ 0x000000 ────────────────────────────────────────────────────────── │ 0x000 – 0x0FF │ CCCR │ 卡通用控制寄存器 │ │ 0x100 – 0x1FF │ FBR Fn1 │ Function 1 基础寄存器 │ │ 0x200 – 0x2FF │ FBR Fn2 │ Function 2 基础寄存器 │ │ ... │ ... │ (Fn3~Fn7 依次排列) │ │ 0x700 – 0x7FF │ FBR Fn7 │ Function 7 基础寄存器 │ │ 0x800 – 0xFFF │ RFU │ 保留 │ │ 0x1000–0x17FFF│ CIS Area │ 公共 CIS + 各 Function 的 CIS │ │ 0x18000–0x1FFFF│ RFU │ 保留 │ 0x01FFFF ──────────────────────────────────────────────────────────

2. CCCR — 卡通用控制寄存器 (0x00–0xFF)

CCCR 允许主机检查和控制每个 Function 的使能、中断、总线宽度和块大小

偏移寄存器名类型关键位说明
0x00SDIO RevisionR/OCCCRx[3:0] · SDIOx[7:4]CCCR 格式版本 & SDIO 规范版本 (0x03=SDIO 2.0)
0x02I/O EnableR/WIOE1–IOE7写 1 使能对应 Function
0x03I/O ReadyR/OIOR1–IOR7Function 是否已准备好工作
0x04Interrupt EnableR/WIEN1–IEN7 · IENM逐一使能中断 + 总中断使能
0x05Interrupt PendingR/OINT1–INT7指示哪个 Function 有挂起中断
0x06I/O AbortW/ORES · AS[2:0]中止进行中的传输 + 复位 I/O
0x07Bus InterfaceR/WCD Disable · Bus Width[1:0]卡检测禁止 · 1-bit(00) 或 4-bit(10)
0x08Card CapabilityR/O4BLS · LSC · E4MI · S4MI · SBS · SRW · SMB · SDC低速/多块/挂起/读等待/直接命令能力

3. FBR — 功能基础寄存器 & CIS — 卡信息结构

FBR 快速标识每个 Function 的能力 · CIS 以 Tuple 链表形式提供详细的卡和功能描述

FBR 结构 (每个 Function 256B)

Fn n 基址 = n × 0x100

0x00: 标准功能接口码 + CSA 能力
0x01: 扩展接口码 (当 0x00=0x0F 时)
0x02: 电源支持 (EPS/SPS 位)
0x09–0x0B: Function CIS 指针 (24-bit)
0x0C–0x0E: CSA 指针 (Code Storage Area)
0x10–0x11: 此 Function 的 I/O 块大小

CIS — Tuple 链表结构

基于 PCMCIA/Card16 metaformat

每个 Tuple：{TPL_CODE(1B) | TPL_LINK(1B) | Data[N]}
0x15 CISTPL_VERS_1：版本/产品字符串
0x20 CISTPL_MANFID：制造商 ID (JEDEC)
0x21 CISTPL_FUNCID：功能标识
0x22 CISTPL_FUNCE：功能扩展数据
0x91 CISTPL_SDIO_STD：SDIO 标准标志
0xFF CISTPL_END：链结束标记

Linux 内核中的解析路径
mmc_attach_sdio() (core/sdio.c) → sdio_read_cccr() → sdio_read_common_cis() → sdio_read_func_cis() (core/sdio_cis.c)
解析结果存入 sdio_func 的 vendor、device、class、max_blksize 等字段，构造 sdio_device_id 用于驱动匹配。

4. Linux SDIO 驱动架构

SDIO 在 MMC 子系统中的三层位置 · 核心源文件 · sdio_func 与 sdio_driver 的匹配机制

Host 层

drivers/mmc/host/

硬件控制器驱动，实现 mmc_host_ops。SDIO 特有回调：enable_sdio_irq（使能 SDIO 中断）、ack_sdio_irq（应答）。
SDHCI 标准控制器天然支持 SDIO 中断。

Core 层

drivers/mmc/core/sdio*.c

sdio.c：卡检测 (CMD5) + 初始化
sdio_cis.c：CIS Tuple 解析
sdio_bus.c：sdio_bus_type 总线 + 设备/驱动匹配
sdio_io.c：I/O 辅助函数
sdio_irq.c：SDIO 中断分发
sdio_ops.c：底层 CMD52/53 封装

Function Driver 层

drivers/net/wireless/ 等

SDIO 函数驱动不放在 drivers/mmc/ 下，而是放在各自子系统目录。
brcmfmac (Broadcom WiFi)
mwifiex (Marvell WiFi)
ath6kl (Atheros WiFi)
通过 sdio_register_driver() 注册到 sdio_bus。

/* ===== SDIO 初始化流程 — 从卡检测到驱动 probe ===== */ mmc_rescan() └── mmc_attach_sdio() // CMD5 → R4 响应 → 确认 SDIO 卡 ├── mmc_send_io_op_cond() // CMD5 查询 I/O OCR ├── mmc_sdio_init_card() // 读 CCCR → CIS → 初始化每个 Function │ ├── sdio_read_cccr() // 读 CCCR 0x00–0x13 │ ├── sdio_read_common_cis() // 解析公共 CIS Tuples │ ├── sdio_read_func_cis(fn) // 解析每个 Function 的 CIS │ └── sdio_init_func(fn) // 设置块大小、使能 Function └── mmc_add_card() // 注册 card → sdio_bus 扫描 └── sdio_bus_match() // vendor/device/class 三元组匹配 └── sdio_driver.probe(func, id) // 函数驱动入口

5. SDIO 函数驱动开发 — 完整模板

sdio_driver 注册 · probe/remove · CMD52/CMD53 I/O 操作 · 中断处理 · 主机锁定

/* ===== 1. 设备 ID 表 + MODULE_DEVICE_TABLE ===== */ static const struct sdio_device_id my_ids[] = { { SDIO_DEVICE(0x02d0, 0xa9a6) }, // vendor=device, 匹配单个产品 { SDIO_DEVICE_CLASS(SDIO_CLASS_WLAN) }, // 匹配所有 WLAN 类 SDIO 设备 { }, // 终止标记 }; MODULE_DEVICE_TABLE(sdio, my_ids); /* ===== 2. probe — 函数被检测到时调用 ===== */ static int my_probe(struct sdio_func *func, const struct sdio_device_id *id) { struct my_dev *priv; int ret; sdio_claim_host(func); // 获取主机 -- 保证原子操作 /* 使能 Function (若尚未使能) */ ret = sdio_enable_func(func); if (ret) goto release; /* 设置 I/O 块大小 */ sdio_set_block_size(func, 512); /* 通过 CMD53 读设备信息 (多字节 burst) */ u8 buf[64]; sdio_memcpy_fromio(func, buf, 0x00, 64); /* 通过 CMD52 写单字节寄存器 */ sdio_writeb(func, 0x01, 0x02, &ret); // 写 0x01 到地址 0x02 sdio_release_host(func); /* 分配私有数据结构 */ priv = kzalloc(sizeof(*priv), GFP_KERNEL); sdio_set_drvdata(func, priv); /* 注册中断处理 */ sdio_claim_irq(func, my_irq_handler); return 0; release: sdio_release_host(func); return ret; } /* ===== 3. remove — 设备拔出或驱动卸载 ===== */ static void my_remove(struct sdio_func *func) { struct my_dev *priv = sdio_get_drvdata(func); sdio_release_irq(func); // 注销中断 sdio_claim_host(func); sdio_disable_func(func); // 关闭 Function sdio_release_host(func); kfree(priv); } /* ===== 4. 中断处理 — 运行在中断上下文 ===== */ static void my_irq_handler(struct sdio_func *func) { struct my_dev *priv = sdio_get_drvdata(func); u8 status; int err; sdio_claim_host(func); // IRQ 中也需要 claim status = sdio_readb(func, MY_INT_STATUS_REG, &err); sdio_writeb(func, status, MY_INT_CLEAR_REG, &err); // W1C 清除 sdio_release_host(func); if (status & MY_RX_READY) schedule_work(&priv->rx_work); // 推迟到 workqueue } /* ===== 5. sdio_driver 结构体 + 注册 ===== */ static struct sdio_driver my_driver = { .name = "my_sdio", .id_table = my_ids, .probe = my_probe, .remove = my_remove, }; module_driver(my_driver, sdio_register_driver, sdio_unregister_driver);
I/O 操作底层命令用途数据量
sdio_readb/writebCMD52读写单字节 (CCCR/FBR/配置寄存器)1 字节
sdio_memcpy_fromio/toioCMD53 (incr addr)Burst 读写 (递增地址 — 数据块)≤ block_size × N
sdio_readsb/writesbCMD53 (fixed addr)FIFO 读写 (固定地址 — 数据端口)≤ block_size × N
sdio_f0_readb/writebCMD52 (Fn0)强制操作 Function 0 (CCCR 区域)1 字节
⚠️ 关键注意点
claim_host: 任何需要原子性保证的多步 I/O 序列必须包裹在 sdio_claim_host()/release_host() 之间，包括中断处理函数！
中断上下文: sdio_claim_irq() 的回调在硬中断上下文执行 → 不能睡眠 → 重活必须推后到 workqueue 或 tasklet。
块大小: 务必在 probe 中通过 sdio_set_block_size() 设置合适的值。CMD53 多字节传输受此限制。

6. 主流 SDIO WiFi 驱动

brcmfmac (Broadcom) · mwifiex (Marvell) · ath6kl (Atheros) — 三大 SDIO WiFi 驱动对比

brcmfmac (Broadcom)

drivers/net/wireless/broadcom/brcm80211/

架构: FullMAC — 固件处理全部 802.11 协议，驱动仅负责 SDIO 传输 + cfg80211 接口。
兼容 ID: SDIO_DEVICE(BRCM_VENDOR_ID, xxx)
固件加载: probe 时通过 firmware API 加载 brcmfmacXXXX-sdio.bin + NV 配置文件。
特点: 最常见的树莓派 WiFi 方案 (Cypress CYW43438)，Linux 主线支持最完善。

mwifiex (Marvell)

drivers/net/wireless/marvell/mwifiex/

架构: FullMAC — 固件处理 MLME。
SDIO 入口: mwifiex_sdio_probe() — 在 sdio_bus 上注册。
特点: 同时支持 SDIO + PCIe 接口 (共用核心逻辑)，广泛应用于 Chromebook、Steam Deck。
芯片: 88W8797, 88W8897, 88W8997 (NXP 收购后)。

ath6kl (Atheros/Qualcomm)

drivers/net/wireless/ath/ath6kl/

架构: FullMAC — 固件管理 802.11 协议。
特点: AR6003/AR6004 系列，广泛应用于嵌入式设备。由 ath6kl 演变出的 ath10k 支持 PCIe/USB/SDIO。
注意: 新一代 Qualcomm 方案用 ath11k/ath12k，支持 SDIO 的 WCN 系列芯片。

SoftMAC vs FullMAC — 理解 WiFi 驱动架构
SoftMAC (mac80211)：MLME 在驱动/内核中实现，硬件只做 PHY+部分 MAC。例：Intel iwlwifi、Mediatek mt76。
FullMAC：MLME 完全在固件中，驱动仅负责传输 802.3 帧 + cfg80211 命令。SDIO WiFi 几乎都是 FullMAC — 固件处理复杂的 802.11 状态机，驱动代码相对精简。
FullMAC 的代价：需要加载 vendor 闭源固件，调试 802.11 问题只能看固件日志。
