1. CMOS 图像传感器 — 从光子到数字

光电转换原理 · 像素架构 (3T/4T/5T) · Rolling Shutter vs Global Shutter · BSI/FSI · 堆叠式传感器

像素工作原理

光电二极管 → Floating Diffusion → ADC

① 曝光 (Integration): 光子击中光电二极管 → 产生电子-空穴对 → 电子收集在 PD 中。光强越强 → 电子越多。
② 转移 (Transfer): TX 栅极导通 → 电子从 PD 转移到 Floating Diffusion (FD) 节点。
③ 读出 (Readout): FD 电压变化 = Q/C → 源跟随器 (SF) 缓冲 → 列 ADC 量化 → 数字像素值。
④ 复位 (Reset): RST 栅极导通 → FD 复位到 VDD → 准备下一次曝光。

4T 像素架构 (主流)

Transfer Gate + 相关双采样 (CDS)

4T = 4 个晶体管 (TX/RST/SF/SEL) + 1 个光电二极管。
CDS 相关双采样: 先读复位电压 (暗电平) → 再读信号电压 → 相减消除 kTC 噪声 和 FPN。
3T 像素: 无 TX 栅极 — 噪声大，已淘汰。
5T 像素: 额外晶体管用于 全局快门 (GS) — 每个像素有模拟存储电容，所有像素同时曝光。

Rolling Shutter (RS)

逐行曝光 · 逐行读出

每行像素依次开始积分 → 逐行读出。不同行曝光时刻不同 → 快速运动产生果冻效应 (skew/wobble)。
优点: 像素简单、成本低、填充因子高。
缺点: 运动伪影、LED 闪烁条带。

Global Shutter (GS)

所有像素同时曝光

所有像素同时开始和结束曝光 → 像素内有模拟存储节点暂存电荷 → 逐行读出。
优点: 零运动伪影 — 工业检测/自动驾驶必需。
缺点: 像素复杂 (5T+)、额外存储电容降低填充因子、更高的暗电流。

BSI 背照式 vs FSI 前照式

光从背面进入 vs 正面进入

FSI (传统): 光线穿过金属布线层到达 PD → 反射/吸收损失→ 低 QE (~40-60%)。
BSI (现代): 晶圆翻转减薄 → 光从背面直接照射 PD → 无金属遮挡 → 高 QE (~80-90%)。
堆叠式 (Stacked): PD 层 + 逻辑层分别制造 → Cu-Cu 键合 → 极致性能。

/* ═══ 4T 像素读出时序 (Rolling Shutter) ═══ */ ┌── 曝光时间 ──┐ Row 0: [RST─TX──...──TX─SF─ADC] ← 先复位, 曝光, 再转移读出 Row 1: [RST─TX──...──TX─SF─ADC] Row 2: [RST─TX──...──TX─SF─ADC] ... ← 逐行错开 = Rolling Shutter /* ═══ 5T 像素 (Global Shutter) ═══ */ All Rows: [同时 RST─TX──存储到 Memory Node] ← 全局同时曝光 Row 0: [SF─ADC] ← 逐行读出存储的电荷 Row 1: [SF─ADC] ... ← 无运动伪影!

量子效率 · 信噪比 · 动态范围 · 满阱容量 · 读出噪声 · 暗电流 · 光电响应非均匀性

噪声源模型

总噪声 = √(σ²_photon + σ²_dark + σ²_read + σ²_FPN)

光子散粒噪声 (Photon Shot Noise): 光的量子本质 → 光子到达时间的泊松分布 → σ² = S (信号电子数)。物理极限 — 无法消除。
暗电流散粒噪声: 热生电子的泊松分布 → 长曝光 + 高温时显著。
读出噪声: 像素源跟随器 + ADC 的热噪声和 1/f 噪声 → 暗场景的 SNR 瓶颈。
固定模式噪声 (FPN): 像素间增益/偏移差异 → CDS 消除 offset FPN，但 gain FPN 需 ISP 校正。

SNR 曲线 — 光强 vs 信噪比

三个区域: 读出噪声限制 → 散粒噪声限制 → 饱和

低光区: 信号 < 读出噪声 → SNR 由读出噪声决定 → RN 越低越好。
中光区: 信号 > RN → SNR ∝ √S (散粒噪声主导) → QE 越高 SNR 越高。
饱和区: 信号 = FWC → SNR 达到最大值 SNR_max = 20log₁₀(√FWC)。
提升 SNR 路径: ① 增大像素 (FWC↑) ② 降低 RN ③ 提高 QE。

3. Bayer 彩色滤波阵列 & HDR 技术

Bayer CFA → 去马赛克 · RGB-IR · RCCB · 多重曝光 HDR · DCG · LOFIC · 2024 前沿

Bayer Pattern (RGGB)

2×2 重复单元 — 50% G, 25% R, 25% B

设计哲学: 人眼对绿色最敏感 → 绿色像素数加倍。
常见变体: RGGB (最常用)、BGGR、GBRG、GRBG (起始位置不同)。
去马赛克 (Demosaic): 从 Bayer RAW 恢复全分辨率 RGB → 双线性/边缘自适应/深度学习。
Quad Bayer (4-cell): 2×2 同色像素 → 暗光下合并为 1 个大像素 (2×2 binning) → 亮光下全分辨率 Remosaic。

HDR — 120dB 动态范围

三种主流 HDR 技术路线

多重曝光 (eHDR): 长/中/短曝光各一帧 → ISP 合成 → 运动伪影 (鬼影)。
DCG (双转换增益): 单次曝光中切换 FD 电容 → HCG (高增益, 暗部) + LCG (低增益, 亮部) → 无运动伪影。
LOFIC: 横向溢出积分电容 → PD 饱和后电荷溢入额外电容 → 单帧 130dB。
LI-HDR: 行交错长短曝光 → 单帧 HDR，垂直分辨率部分损失。

4. 曝光三要素与 3A 算法

光圈 (Aperture) · 快门 (Shutter) · 增益 (Gain/ISO) · AE/AWB/AF 的基本原理

曝光三要素

亮度 = 光圈 × 快门 × 增益

光圈 (Aperture): 镜头通光孔径 → 影响景深和进光量。固定光圈镜头 (手机/嵌入式) → 不可调。
快门 (Exposure Time): 积分时间 → 电子收集时长。范围: ~10μs ~ 1s。
增益 (Gain / ISO): 模拟增益 (放大器) + 数字增益 (乘法) → 放大信号也放大噪声。
三者共同决定曝光量 — AE 自动曝光算法 的目标是协调三者使图像达到目标亮度。

AE — 自动曝光

目标亮度 → 调整积分时间 + 增益

测光: 统计图像亮度直方图 → 计算均值/中值/加权均值。
策略: ① 优先延长积分时间 (降低噪声) ② 积分时间够长后仍暗 → 加增益 ③ 太亮 → 先减增益再减时间。
防闪烁 (Anti-Flicker): 检测 50/60Hz 频闪 → 积分时间锁定为电网周期的整数倍 (10ms/8.33ms)。

AWB — 自动白平衡

消除光源色温偏色

灰度世界假设: 场景的平均颜色是灰色 → R/G/B 均值应相等 → 计算增益使 R=B=G。
色温估计: 在 R/G vs B/G 色彩空间中定位最接近的黑体辐射曲线点。
常用方法: 灰度世界 + 完美反射 + 色温映射 + 机器学习。

5. Linux V4L2 传感器驱动开发

v4l2_subdev · I2C/CCI 寄存器配置 · 模式表 (Mode Table) · Device Tree 绑定 · 异步注册

传感器驱动架构

v4l2_subdev_ops — pad · video · core

I2C/CCI 控制: 传感器通过 I2C 或 CCI 接口配置寄存器 (分辨率/帧率/曝光/增益/MIPI lane 数)。
v4l2_subdev: 注册为 V4L2 子设备，提供 pad ops (get_fmt/set_fmt)、video ops (s_stream/g_frame_interval)、core ops (s_power)。
模式表: 每个传感器有一组预定义的模式 (分辨率+帧率+link_freq+像素格式组合) — 驱动在 enum_frame_size 中遍历。

V4L2 Async 异步注册

跨总线 probe 顺序协调

传感器 (I2C) 和 CSI 控制器 (Platform) 独立 probe → V4L2 Async 框架 通过 v4l2_async_notifier 协调:
① Sensor probe → v4l2_async_register_subdev()
② CSI 控制器 probe → v4l2_async_notifier_register()
③ Notifier 等待 DT 中 remote-endpoint 关联的 sensor subdev 就绪
④ .bound() 回调 → .complete() 回调 (所有子设备就绪)

/* ===== IMX219 传感器驱动 — 关键结构 ===== */ static const struct imx219_mode supported_modes[] = { { /* 3280x2464 @ 15fps, 2-lane MIPI */ .width = 3280, .height = 2464, .code = MEDIA_BUS_FMT_SRGGB10_1X10, // Bayer RGGB, 10-bit RAW .fps = 15, .link_freq = 456000000, // 456 MHz MIPI 链路频率 .reg_list = mode_3280x2464_regs, // 寄存器序列 (数百条 I2C 写) }, { /* 1920x1080 @ 30fps */ .width = 1920, .height = 1080, .code = MEDIA_BUS_FMT_SRGGB10_1X10, .fps = 30, .link_freq = 456000000, .reg_list = mode_1920x1080_regs, }, }; /* ===== V4L2 subdev 操作 — pad ops (格式协商) ===== */ static int imx219_set_fmt(struct v4l2_subdev *sd, struct v4l2_subdev_state *state, struct v4l2_subdev_format *fmt) { struct v4l2_mbus_framefmt *mbus_fmt = &fmt->format; /* 从 supported_modes[] 找最匹配的分辨率 */ mode = imx219_find_best_mode(mbus_fmt->width, mbus_fmt->height); mbus_fmt->width = mode->width; mbus_fmt->height = mode->height; mbus_fmt->code = mode->code; return 0; } static const struct v4l2_subdev_pad_ops imx219_pad_ops = { .get_fmt = imx219_get_fmt, .set_fmt = imx219_set_fmt, }; /* ===== s_stream — 启动/停止数据流 ===== */ static int imx219_start_streaming(struct imx219 *priv) { // 写入数百个寄存器配置当前模式 for (i = 0; i < mode->num_regs; i++) imx219_write_reg(priv, mode->reg_list[i].addr, mode->reg_list[i].val); return 0; }

传感器寄存器表 — 驱动的核心数据
一个典型的 8MP 传感器驱动中, 寄存器配置表 占据 ~80% 的代码量。每个分辨率模式对应 200~800 条寄存器写入 — 包括 PLL 时钟配置、像素阵列窗口裁剪、MIPI 时序、模拟电路偏置等。这些值来自传感器厂商的datasheet 和 tuning guide — 驱动开发本质上是将硬件手册翻译为 C 结构体。
