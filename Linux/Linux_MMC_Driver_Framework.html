平台驱动

MMC / eMMC

JEDEC JESD84 标准

嵌入式多媒体卡标准。eMMC 将控制器+NAND Flash 封装一体，手机/平板/IoT 标配存储。

SD / SDIO

SD Association 规范

从 MMC 演进。SD 用于可插拔卡；SDIO 复用总线连接 WiFi/蓝牙/GPS 等 I/O 外设。

内核位置

drivers/mmc/

card/ core/ host/ 三个子目录对应三层。CONFIG_MMC 子系统开关，CONFIG_MMC_BLOCK 块设备。

1. 三层架构模型

经典分层设计 — 每一层职责清晰、接口明确、独立编译为内核模块

Card 层

drivers/mmc/card/ → mmc_block.ko

将存储抽象为标准块设备 /dev/mmcblkX。"mmcqd" 线程取 block request 构造 mmc_request 向下派发。管理 RPMB、Boot、GP 特殊分区。

Core 层

drivers/mmc/core/ → mmc_core.ko

子系统中枢：卡生命周期（mmc_rescan → mmc_init_card）、协议抽象（MMC/SD/SDIO 各有 mmc_bus_ops）、请求派发（mmc_wait_for_req → host->ops->request）、公共工具（mmc_of_parse 解析 DT）。

Host 层

drivers/mmc/host/ → 平台驱动

SoC 硬件适配层，实现 struct mmc_host_ops。常见 IP：SDHCI（Intel/Qualcomm/NVIDIA）、DesignWare MMC（Rockchip/STM32）、Freescale eSDHC（i.MX）、Allwinner SUNXI。

2. 核心数据结构

mmc_host · mmc_card · mmc_ios · mmc_host_ops — 子系统围绕这四个结构体运转

mmc_host

include/linux/mmc/host.h

子系统的根对象。f_min/f_max 时钟范围、caps/caps2 能力位、ops 操作回调、ios 总线状态、card 已挂载卡指针。
分配 → 注册：mmc_alloc_host(extra, dev) → mmc_add_host(host) → 首次 mmc_rescan()。

mmc_card

include/linux/mmc/card.h

描述已检测的物理卡。raw_cid[4]/raw_csd[4] 原始寄存器 + 解码后的 cid/csd。eMMC 额外含 ext_csd (512B)。type: MMC_TYPE_MMC / _SD / _SDIO。
mmc_alloc_card() → mmc_add_card() → 触发 mmc_blk_probe()。

mmc_ios

I/O 总线电气 & 时序状态

Core 通过 host->ops->set_ios(host, &ios) 通知 Host 更新硬件。
clock: 当前频率   power_mode: OFF→UP→ON
bus_width: 1/4/8   timing: LEGACY→HS→HS200→HS400
signal_voltage: 3.3V/1.8V   drv_type: Type-0~4

mmc_host_ops

Host 必须实现的回调集

必选 — request (I/O 事务)、set_ios (总线配置)
高速 — execute_tuning、start_signal_voltage_switch
DMA — pre_req / post_req (提前映射/延迟解映射)
常用 — get_cd、get_ro、card_busy、hw_reset

mmc_rescan → 三种协议探测 → mmc_init_card 九步 — 从无卡到就绪

或轮询 → mmc_rescan() → power_up
CMD0 idle → CMD8

① SDIO 探测

CMD5 (IO_SEND_OP_COND) → 若响应则初始化 SDIO。典型设备：WiFi/蓝牙模块。需 DT 配 keep-power-in-suspend。

5. 总线速度模式

从 Legacy 26 MB/s 到 HS400 400 MB/s — eMMC 速度等级演进全景

模式速率电压总线时钟带宽×8引入

High Speed SDRSDR3.3/1.8V4,8≤52 MHz52 MB/sMMC 4.1
High Speed DDR (DDR52)DDR3.3/1.8V4,8≤52 MHz104 MB/sMMC 4.3
HS200SDR1.8V4,8≤200 MHz200 MB/seMMC 4.5
HS400DDR1.8V8≤200 MHz400 MB/seMMC 5.0

HS200 (eMMC 4.5+)

SDR 200 MHz → 200 MB/s

必须 1.8V。需要 Tuning：Host 通过 CMD21 发送已知数据块，校准内部采样点补偿信号往返延迟。写入源同步，读取需调谐。驱动强度 Type-0~3。

HS400 (eMMC 5.0+)

DDR 200 MHz → 400 MB/s

必须 8-bit + 1.8V。新增 Data Strobe (DS)：设备读取时生成 DS 信号，Host 利用其边沿可靠捕获 DDR 数据。须先完成 HS200 调谐。额外 Type-4 驱动 (40Ω)。

驱动阻抗相对说明驱动阻抗相对说明
Type-050Ω×1.0默认，必选Type-266Ω×0.75最弱 200MHz
Type-133Ω×1.5强驱动长走线Type-3100Ω×0.5低 EMI/噪声
Type-440Ω×1.2HS400 专用
⚠️ HS400 切换顺序
HS200(Tuning) → 先 HS_TIMING=0x01 → 再降频≤52M → DDR 8-bit → HS_TIMING=0x03
HS200 可跑低速，但 High Speed 不能超 52MHz。先降频再切时序 → 非法状态 → 命令失败。

6. Host 驱动开发

request 回调 — 中断处理 — 状态机 — probe 最小模板

request — 事务入口

host->ops->request(host, mrq)

① get_cd() 检查卡存在 → 失败则 -ENOMEDIUM
② 控制器忙 → list_add_tail 排队
③ 有数据 → dma_map_sg() + 写 BLKSZ/BLKCNT 寄存器
④ 根据 cmd->flags 配置响应类型 → 写 CMD/ARG → 启动。后续在中断中异步完成。

中断处理 — 异步完成

CMD_DONE → DATA_DONE → STOP → done()

CMD_DONE: 读 RSP0~3，超时→-ETIMEDOUT，CRC→-EILSEQ
DATA_DONE: dma_unmap_sg()，统计 bytes_xfered
STOP: 多块传完发 CMD12
最后 mmc_request_done(host->mmc, mrq) → 出队下一个。

/* ===== request 回调 — 核心骨架 ===== */ static void xxx_mmc_request(struct mmc_host *mmc, struct mmc_request *mrq) { struct xxx_mmc_host *host = mmc_priv(mmc); if (!xxx_mmc_get_cd(mmc)) { // 卡拔出 → 立即失败 mrq->cmd->error = -ENOMEDIUM; mmc_request_done(mmc, mrq); return; } spin_lock(&host->lock); if (host->state != STATE_IDLE) { // 忙 → 排队等待 list_add_tail(&mrq->node, &host->queue); spin_unlock(&host->lock); return; } host->mrq = mrq; host->state = STATE_SENDING_CMD; spin_unlock(&host->lock); if (mrq->data) // DMA 准备 xxx_mmc_setup_data(host, mrq->data); xxx_mmc_prepare_cmd(host, mrq->cmd); // 写 CMD/ARG/响应类型 xxx_mmc_start_cmd(host); // 启动 → 使能 IRQ } /* ===== 中断处理 ===== */ static irqreturn_t xxx_mmc_irq(int irq, void *devid) { struct xxx_mmc_host *host = devid; u32 st = readl(host->base + MMC_STATUS); struct mmc_command *cmd = host->mrq->cmd; struct mmc_data *data = host->mrq->data; if (st & CMD_DONE) { // 命令完成 for (int i = 0; i < 4; i++) cmd->resp[i] = readl(host->base + RSP + i*4); if (st & CMD_TIMEOUT) cmd->error = -ETIMEDOUT; else if (st & CMD_CRC_ERR) cmd->error = -EILSEQ; if (data && !cmd->error) return IRQ_HANDLED; // 等 DATA_DONE goto finish; } if (st & DATA_DONE) { // 数据传输完成 dma_unmap_sg(...); data->bytes_xfered = data->blocks * data->blksz; if (st & DATA_TIMEOUT) data->error = -ETIMEDOUT; else if (st & DATA_CRC_ERR) data->error = -EILSEQ; if (host->mrq->stop && !data->error) { // 多块 → STOP xxx_mmc_send_stop(host, host->mrq->stop); return IRQ_HANDLED; } goto finish; } finish: host->state = STATE_IDLE; mmc_request_done(host->mmc, host->mrq); // 通知 Core xxx_mmc_process_next(host); // 出队 return IRQ_HANDLED; }
IDLE → SENDING_CMD → CMD_DONE → 有数据? XFER → DATA_DONE → 多块? STOP → done()
最小 probe / remove 模板
static int xxx_mmc_probe(struct platform_device *pdev) { struct xxx_mmc_host *host; struct mmc_host *mmc; mmc = mmc_alloc_host(sizeof(*host), &pdev->dev); // ① 分配 if (!mmc) return -ENOMEM; host = mmc_priv(mmc); host->mmc = mmc; host->base = devm_platform_ioremap_resource(pdev, 0); // ② 资源 host->irq = platform_get_irq(pdev, 0); host->clk = devm_clk_get(&pdev->dev, "mmc"); mmc->ops = &xxx_mmc_ops; // ③ 属性 mmc->f_min = 400000; mmc->f_max = 50000000; mmc->ocr_avail = MMC_VDD_32_33 | MMC_VDD_33_34; mmc->caps = MMC_CAP_4_BIT_DATA | MMC_CAP_8_BIT_DATA | MMC_CAP_MMC_HIGHSPEED; mmc->max_segs = 64; mmc->max_seg_size = 65536; mmc_of_parse(mmc); // ④ DT devm_request_irq(&pdev->dev, host->irq, xxx_mmc_irq, 0, // ⑤ IRQ mmc_hostname(mmc), host); mmc_add_host(mmc); // ⑥ 注册 return 0; } static int xxx_mmc_remove(struct platform_device *pdev) { struct xxx_mmc_host *host = platform_get_drvdata(pdev); mmc_remove_host(host->mmc); mmc_free_host(host->mmc); return 0; }
SDHCI 标准框架 — 优先基于它开发
SD Host Controller Interface (SDA 标准寄存器接口)。内核 sdhci.c 提供完整 mmc_host_ops，平台只需通过 sdhci_ops 覆盖 set_clock / reset / set_uhs_signaling 等 SoC 特定回调。不兼容 SDHCI 的控制器（如 DesignWare MMC）才需从零实现。

Suspend / Resume

系统休眠处理

eMMC: Flush → CMD5 SLEEP → 可选断电 → Resume 发 CMD5 AWAKE
SD 卡: mmc_remove_card → 断电 → 重新枚举
SDIO+WiFi: 保持供电，仅挂起功能驱动，支持 WoWLAN。

电压切换

3.3V ↔ 1.8V

HS200/400 要求 1.8V。流程：CMD6 通知卡 → regulator_set_voltage(vqmmc, 1.8V) → 等 5ms → CMD13 验证。失败则回退低速。

Runtime PM

空闲自动 gating

Host 实现 runtime_suspend/resume，Core 在空闲超 autosuspend_delay 后自动关时钟+断电。I/O 到达时自动恢复。

mmc_of_parse() 注意事项
该函数会覆盖 host->caps/caps2 中与 DT 对应的位。请在 mmc_of_parse() 之前 设代码默认值，之后 设 DT 无法表达的 SoC 私有限制。别名序号通过 of_alias_get_id(np, "mmc") 获取。
