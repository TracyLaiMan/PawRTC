UART — 异步串行

RX · TX · RTS · CTS

通用异步收发器。点对点全双工，无需时钟线 — 收发双方约定相同波特率。每字节以 Start bit + Data bits + Parity + Stop bits 成帧。Linux 通过 /dev/ttyS* 或 /dev/ttyUSB* 访问。

I2C — 同步两线

SCL · SDA · 7/10-bit 地址

内部集成电路总线。多主多从、半双工、仅需 2 根线 (时钟+数据)。开漏输出 + 上拉电阻实现线与。支持 Standard (100kHz) / Fast (400kHz) / Fast+ (1MHz) / High-Speed (3.4MHz)。

SPI — 同步四线

SCLK · MOSI · MISO · CS/SS

串行外设接口。主从全双工、高速 (<50MHz)。4 根线: 时钟 + 主出从入 + 主入从出 + 片选。支持 4 种时钟极性和相位组合 (SPI Mode 0/1/2/3)。无标准协议层 — 纯数据移位。

2. UART — 通用异步收发器

异步串行通信 · 波特率 · 数据帧格式 · RS-232/485 物理层 · Linux TTY 子系统

硬件特性

异步 · 无时钟 · 点对点

工作原理: 收发双方约定相同的波特率，接收方在 Start bit 的下降沿同步自己的采样时钟。每位持续 1/baud_rate 秒。
关键参数: 波特率 (9600/115200/921600/...)、数据位 (5~9, 通常 8)、校验位 (None/Even/Odd)、停止位 (1/1.5/2)。
电气层: TTL UART (0~3.3V/5V) → MCU 直连 · RS-232 (±12V, DB9) → PC 串口 · RS-485 (差分, 多点) → 工业控制。

数据帧格式

每字节的完整封装

空闲: 线路保持高电平 (Mark)
Start Bit: 1 位低电平 (Space) — 同步起点
Data Bits: 5~9 位，LSB 先发
Parity Bit (可选): 奇偶校验
Stop Bit(s): 1/1.5/2 位高电平 — 回到空闲
例: 8N1 = 8 数据位 + No 校验 + 1 停止位 (最常见配置)

硬件流控 — RTS/CTS

防止接收缓冲区溢出

RTS (Request To Send): 发送方→接收方，"我准备好发送了"
CTS (Clear To Send): 接收方→发送方，"可以发送数据了"
当接收方缓冲区接近满 → 拉低 CTS → 发送方暂停。
软件流控: XON (0x11) / XOFF (0x13) 字符在数据流中传输，不适合二进制数据。

/* ===== Linux UART — TTY 设备 API ===== */ #include <termios.h> // POSIX 终端控制接口 #include <fcntl.h> int fd = open("/dev/ttyS0", O_RDWR | O_NOCTTY); /* 获取/设置串口参数 */ struct termios tty; tcgetattr(fd, &tty); /* 波特率 */ cfsetispeed(&tty, B115200); // 输入波特率 cfsetospeed(&tty, B115200); // 输出波特率 /* 数据帧: 8N1 */ tty.c_cflag &= ~PARENB; // 无校验 tty.c_cflag &= ~CSTOPB; // 1 停止位 tty.c_cflag &= ~CSIZE; // 清除数据位 tty.c_cflag |= CS8; // 8 数据位 /* 硬件流控 */ tty.c_cflag |= CRTSCTS; // 启用 RTS/CTS /* 原始模式 (禁用行缓冲、回显等) */ cfmakeraw(&tty); tcsetattr(fd, TCSANOW, &tty); // 立即生效 /* 读写 */ write(fd, "AT\r\n", 4); read(fd, buf, sizeof(buf));
Linux TTY 子系统 — UART 的内核实现
硬件层: UART 控制器驱动 (drivers/tty/serial/) — 实现 struct uart_ops (tx_empty/set_mctrl/get_mctrl/stop_tx/start_tx...)
TTY 核心层: drivers/tty/tty_io.c — 线路规程 (Line Discipline) 管理，提供 POSIX termios 接口
用户空间 /dev/ttyS* (8250/16550A 兼容 UART) 或 /dev/ttyAMA* (ARM AMBA PL011)

3. I2C — 内部集成电路总线

同步两线 · 多主多从 · 开漏 + 上拉 · SCL/SDA · Linux I2C 子系统 · regmap 抽象

硬件特性

开漏输出 + 上拉电阻 — 线与逻辑

SCL (Serial Clock): 时钟线 — 主设备驱动
SDA (Serial Data): 数据线 — 双向
开漏 (Open-Drain): 设备只能拉低或释放 (高阻态)。上拉电阻 (通常 2.2kΩ~10kΩ) 将空闲总线拉高。
线与 (Wired-AND): 任意设备拉低 SDA 即可拉低整条总线 → 实现多主仲裁和 ACK 应答
Clock Stretching: 从设备可在应答后拉低 SCL，暂停主设备时钟 — 给自己更多处理时间。

I2C 事务时序

START → 地址+R/W → ACK → 数据 → STOP

① START: SCL 高时 SDA 下降沿
② Address + R/W: 7-bit 从地址 + 1-bit 方向 (0=写/1=读)
③ ACK: 第 9 个时钟脉冲=从设备拉低 SDA (NACK=高电平)
④ Data: 8-bit 数据 + ACK/NACK (可多字节连续传输)
⑤ STOP: SCL 高时 SDA 上升沿
⑥ Repeated START: 不释放总线直接发新 START — 用于原子性"写寄存器地址→读数据"操作

/* ===== I2C 设备驱动 — 从设备侧 (probe + regmap) ===== */ #include <linux/i2c.h> #include <linux/regmap.h> static const struct of_device_id my_i2c_ids[] = { { .compatible = "vendor,my-sensor" }, { } }; static int my_i2c_probe(struct i2c_client *client) { struct regmap *map; int chip_id; /* Regmap — 现代 I2C 驱动的标准抽象层 */ map = devm_regmap_init_i2c(client, &my_regmap_config); // Regmap 自动处理: I2C 读写封装 · 寄存器缓存 · 锁保护 · debugfs /* 读芯片 ID 寄存器 */ regmap_read(map, 0x00, &chip_id); // reg 0x00 → chip_id if (chip_id != 0x5A) return -ENODEV; /* 写配置寄存器 */ regmap_write(map, 0x01, 0x03); // reg 0x01 = 0x03 /* 批量更新 (对多寄存器原子操作) */ regmap_update_bits(map, 0x02, BIT(3), BIT(3)); // 置位 reg 0x02 bit 3 return 0; } static struct i2c_driver my_driver = { .driver = { .name = "my-sensor", .of_match_table = my_i2c_ids }, .probe = my_i2c_probe, .id_table = my_i2c_id, // 非 DT 平台的传统匹配表 }; module_i2c_driver(my_driver); /* ===== 设备树 I2C 绑定 ===== */ &i2c0 { clock-frequency = <400000>; // 400kHz Fast Mode my_sensor@48 { compatible = "vendor,my-sensor"; // 匹配驱动 reg = <0x48>; // 7-bit I2C 地址 vdd-supply = <&vcc_3v3>; // 电源 interrupt-parent = <&gpio0>; interrupts = <12 IRQ_TYPE_EDGE_FALLING>; // 中断引脚 }; };
Regmap — 通用寄存器映射抽象层
Regmap 是内核提供的统一寄存器访问接口，抽象了 I2C/SPI/MMIO 的底层差异。同一份 regmap 代码可通用于不同总线。提供: 寄存器缓存 (flat/rbtree)、锁保护、debugfs 寄存器 dump (/sys/kernel/debug/regmap/)、批量读写优化。现代 I2C 驱动应优先使用 regmap 而非裸 I2C API。

4. SPI — 串行外设接口

同步四线全双工 · SCLK/MOSI/MISO/CS · 4 种时钟模式 · 无协议层 — 纯数据移位 · Linux spi子系统

硬件特性

全双工 · 推挽驱动 · 主控时钟

SCLK (Serial Clock): 主设备生成，从设备接收
MOSI (Master Out Slave In): 主→从数据
MISO (Master In Slave Out): 从→主数据
CS/SS (Chip Select / Slave Select): 主设备拉低选中从设备 (通常低有效)
推挽输出 (非开漏) → 信号质量好，速率高。无寻址协议 — 靠 CS 硬件片选。全双工: MOSI 和 MISO 同时移位 — 读和写在同一时钟周期完成。

SPI Mode — 4 种 CPOL/CPHA 组合

时钟极性 (CPOL) + 时钟相位 (CPHA)

Mode 0 (CPOL=0, CPHA=0): 空闲低, 上升沿采样 ★ 最常见
Mode 1 (CPOL=0, CPHA=1): 空闲低, 下降沿采样
Mode 2 (CPOL=1, CPHA=0): 空闲高, 下降沿采样
Mode 3 (CPOL=1, CPHA=1): 空闲高, 上升沿采样
主从双方必须使用相同 Mode，否则数据错位。

/* ===== SPI 设备驱动 — 从设备侧 ===== */ #include <linux/spi/spi.h> static int my_spi_probe(struct spi_device *spi) { u8 tx_buf[4] = {0x9F, 0x00, 0x00, 0x00}; // 读 ID 命令 u8 rx_buf[4]; /* SPI 传输 — 全双工: MOSI 发送 tx_buf, MISO 同时接收 rx_buf */ struct spi_transfer t = { .tx_buf = tx_buf, // 发送缓冲区 .rx_buf = rx_buf, // 接收缓冲区 .len = 4, // 传输字节数 .speed_hz = 10000000, // 10MHz (可覆盖 DT 默认值) }; struct spi_message msg; spi_message_init(&msg); spi_message_add_tail(&t, &msg); spi_sync(spi, &msg); // 同步传输 — 阻塞等待完成 /* 或者使用 regmap (推荐) */ struct regmap *map = devm_regmap_init_spi(spi, &config); regmap_read(map, 0x00, &id); // Regmap 封装了 SPI 读写 return 0; } static struct spi_driver my_driver = { .driver = { .name = "my-spi-dev", .of_match_table = my_ids }, .probe = my_spi_probe, }; module_spi_driver(my_driver); /* ===== 设备树 SPI 绑定 ===== */ &spi0 { cs-gpios = <&gpio0 8 GPIO_ACTIVE_LOW>; // CS 引脚 — 可用 GPIO 代替硬件 CS flash@0 { compatible = "jedec,spi-nor"; // 标准 SPI NOR Flash 驱动 reg = <0>; // CS 编号 (对应 cs-gpios 索引) spi-max-frequency = <50000000>; // 50MHz spi-cpha; // CPHA=1 spi-cpol; // CPOL=1 → Mode 3 }; adc@1 { compatible = "ti,adc128s052"; reg = <1>; // 第二个 CS spi-max-frequency = <2000000>; }; };
SPI vs I2C vs UART — 传输模式对比
SPI: 全双工 — MOSI 和 MISO 同时移位。发送 N 字节同时接收 N 字节响应 (tx_buf/rx_buf 可相同或分离)。
I2C: 半双工 — 先发送地址字节，再在数据阶段读/写。写后读需要 Repeated START。
UART: 全双工 — RX 和 TX 完全独立，可同时收发。异步 — 无需共享时钟。
