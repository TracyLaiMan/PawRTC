总体架构：相机系统的三层模型

任何数字相机都可抽象为三层：光学层（收光）、传感器层（光→电→数）、ISP 层（数→图）

🔬 光学层

Optical Layer

镜头组负责收集并聚焦光线。关键参数：焦距 f、光圈 F/#、视场角 FOV。IR-cut 滤掉近红外以保证色彩准确。这一层的物理限制（衍射、像差、景深）决定了成像的上限。

📷 传感器层

Sensor Layer

像素阵列（光电二极管）+ CFA + ADC。光子通过光电效应转化为电子，再经 ADC 量化为数字信号。读出方式有 Rolling Shutter（逐行）和 Global Shutter（全局）两种。

⚙️ ISP 层

Image Signal Processor

将 RAW Bayer 数据"烹饪"成人眼可接受的彩色图像。15-20 个处理阶段，核心任务包括：去噪、颜色校正、色调映射。同时运行 3A 算法（AE/AF/AWB）闭环控制传感器和镜头。

光学系统：镜头与光路设计

镜头是成像的第一道关口——光圈决定光通量，焦距决定视角，MTF 决定解析力

光圈 (Aperture)

F 值 = f / D

F/2.0 大光圈 → 进光多、景深浅、暗光好
F/8.0 小光圈 → 进光少、景深深、解析力高
每差一档 F，光通量差一倍（F/1.4→F/2.0 差一倍）。

焦距 (Focal Length)

视场角 = 2·arctan(h/2f)

2.8mm 广角（FOV ~110°），安防监控
6mm 标准（FOV ~50°），人脸门禁
12mm+ 长焦（FOV ~25°），车牌识别

景深 (DoF)

Depth of Field

大光圈 + 长焦 → 浅景深 → 背景虚化
小光圈 + 广角 → 深景深 → 全景清晰
定焦镜头通过超焦距最大化景深范围。

MTF 曲线——镜头成像锐度的标尺
MTF（Modulation Transfer Function）反映镜头在不同空间频率（线对/mm）下保留对比度的能力。MTF50（对比度降至 50% 时的频率）是常用评价指标——例如一颗镜头 MTF50 = 150 lp/mm，意味着它能清晰分辨 Sensor 上 3.3μm 的细节（1/(2×150)=3.3μm）。镜头 MTF 与 Sensor 像素尺寸的匹配决定了系统分辨率上限。
红外截止滤光片 (IR-Cut Filter)
Sensor 的硅基光电二极管对近红外光（700-1100nm）非常敏感。如果不滤除，红外成分会让绿色通道偏亮、画面泛红/泛白。IR-Cut 滤掉 650nm 以上的光，但 IR-Cut 可机械切换（ICR）——白天用 IR-Cut，夜晚切换到全光谱 + 红外补光实现夜视。

CMOS 图像传感器 (CIS) 的截面结构，光从上往下穿过各层到达光电二极管

CCD — 电荷耦合器件

Charge-Coupled Device

电荷逐行"搬运"到统一的读出放大器。全局快门天然，噪声极低。但功耗高、速度慢、难以片上集成 ISP。现代应用中几乎被 CMOS 完全取代，仅在天文、科学成像领域仍有使用。

CMOS — 互补金属氧化物半导体

CMOS Image Sensor (CIS)

每个像素自带放大器，片上集成 ADC（列并行架构）。功耗低、速度快、成本低、可片上 ISP。主流技术：FSI（前照式）→ BSI（背照式）→ Stacked（堆栈式）。手机、安防、车载全覆盖。

Rolling Shutter 卷帘快门

逐行曝光 + 逐行读出

每行曝光起始时刻不同，导致果冻效应（Jello Effect）——快速运动物体倾斜或扭曲。优势是像素电路简单、读出噪声更低。消费类 CMOS Sensor 默认模式。

Global Shutter 全局快门

同时曝光 + 逐行读出

所有像素同一时刻开始和结束曝光，无果冻效应。但每个像素需要额外的存储电容（Sample-and-Hold），增加噪声和成本。用于高速运动场景：工业检测、无人机、自动驾驶。

Bayer 模式与颜色滤波阵列 (CFA)

每个像素只感一种颜色——这就是为什么 Sensor 输出的 RAW 数据需要 ISP 重建全彩

Bayer RGGB — 最经典的 CFA

Bryce Bayer, Kodak 1976

2×2 重复单元：G 占 50%、R 25%、B 25%。G 通道翻倍是因为人眼对亮度（绿色贡献最大）最敏感。几乎所有消费级 Sensor 都用 Bayer 或其变体。

其他 CFA 模式

RCCB / RGB-IR / Monochrome

RCCB：一个 C(Clear) 替代一个 G → 低照度更好、但颜色重建更难
RGB-IR：一个 G 换 IR → 同时获取彩色+红外
Quad Bayer：4个同色像素一组 → 像素合并提升灵敏度

CFA 类型2×2 单元特点适用场景
Bayer RGGBR G / G B最成熟，G 占比 50%，色彩还原稳定手机、安防、车载、消费电子
Bayer GRBGG R / B GRGGB 的排列变体，处理类似部分 Sony/OmniVision Sensor
RCCBR C / C B低照度 SNR ↑6dB，但色彩恢复依赖复杂算法暗光安防 (星光级)、车载夜视
Quad Bayer4×4 像素合并亮光: 高分辨率 / 暗光: 4合1 大像素高端手机 (Samsung HM, Sony IMX)
Monochrome无 CFA灵敏度极高（无颜色滤光损失），但无色彩工业检测、天文、结构光
为什么 Bayer 模式 G 通道要占 50%？
人眼视网膜中感知亮度的视杆细胞远多于感知颜色的视锥细胞，对亮度细节的敏感度远高于颜色。在标准亮度公式中：Y = 0.299R + 0.587G + 0.114B，绿色贡献了约 59% 的亮度信息。因此 Bayer 模式将 50% 的像素分配给 G 通道，最大化感知分辨率的同时保持色彩可恢复。

红外夜视：ICR 切换与 IR 补光技术

IPC 安防的核心场景——白天彩色、夜晚黑白红外。ICR（IR-Cut Removal）是连接两个世界的机械开关

ICR

ICR 的工作原理

IR-Cut Filter Removable — 机械切换红外滤光片

ICR 是一个电磁/电机驱动的机械装置，包含两片滤光片：IR-Cut 滤光片（白天模式，滤掉 650nm 以上近红外）和全透玻璃片（夜晚模式，让红外光通过）。当光照降到阈值以下时，电磁铁将 IR-Cut 片移开，切换为全透模式——此时 Sensor 可接收红外补光灯照射物体后反射回来的红外光。切换动作通常耗时 200-500ms，期间画面会有短暂的模糊/色偏。

850nm IR LED

最常用，有可见红曝

波长 850nm 的近红外 LED 是安防相机的标配。人眼不可见红外光，但 LED 本身有微弱的红色光晕（红曝）——在完全黑暗中人眼可以看到淡淡的红光点。优点是 Sensor 在此波段灵敏度较高，照射距离远。

940nm IR LED

无红曝，隐蔽监控

波长 940nm 完全在人眼可见范围之外——零红曝，适合隐蔽监控场景。但 Sensor 在 940nm 的量子效率远低于 850nm（约一半），同等功率下照射距离更短。需要更高功率的 LED 或更灵敏的 Sensor 来补偿。

智能 IR (Smart IR)

动态调节 IR LED 亮度

固定功率 IR 的问题：近距离物体过曝（一片白），远距离亮度不足。Smart IR 根据画面的 AE 统计数据动态调节 IR LED 的 PWM 占空比——当有人靠近相机时自动降低 IR 亮度，避免人脸过曝丢失细节；人走远时恢复全功率。

⚠️ IPC 红外夜视的三大实战问题
1. IR 反射（IR Reflection）——半球/球机护罩、近处墙壁、树叶等近距离物体将 IR 光反射回镜头，造成画面局部过曝（白雾状）。解决：调整 IR LED 角度使光路偏离镜头视轴、使用外置 IR 灯分离光路。
2. ICR 切换的临界振荡——晨昏时刻光照在阈值附近波动，ICR 频繁切换（咔嗒声 + 画面反复变色）。解决：软件中加入滞回（Hysteresis）——例如光照低于 2 lux 切换夜视，但需要高于 5 lux 才切回白天。
3. 红外偏焦（IR Focus Shift）——红外光波长不同于可见光，镜头的折射率不同，导致红外模式下焦点偏移。高端镜头有IR 校正镀膜或共焦设计；普通镜头在切换到夜视后画面会略微变软。

微光全彩：星光级低照度技术

不需要红外补光也能在月光/星光下输出彩色图像——这是 IPC 画质分级的核心指标

什么是"星光级"？

Starlight / Ultra-Low-Light

行业约定：Sensor 在 0.001 lux 及以下 照度仍能输出可辨识的彩色图像，即为"星光级"。参考：满月约 0.1 lux、无月星空约 0.001 lux、阴云密布约 0.0001 lux。星光级 Sensor 在月光下就能全彩成像，无需切换到 IR 黑白模式——这是微光全彩的技术基础。

核心指标：SNR1s

信噪比 = 1 时的最小照度

衡量 Sensor 低照度能力的最硬核指标：在什么照度下画面信噪比降为 1（信号和噪声一样大）。Sony STARVIS 2 的 SNR1s 可达 0.0003 lux（彩色），远超上一代。但 SNR1s 是芯片级指标——实际系统中镜头（F 值）和 ISP 降噪能力同样决定最终低照度画质。

大像素策略

2.0μm → 2.9μm → 3.0μm+

单个像素面积越大 → 捕获光子越多 → 暗光 SNR 越高。星光级 Sensor 坚持使用大像素：Sony IMX662（2.9μm，200万像素星光）、OmniVision OS08A（2.0μm BSI）。手机 Sensor 走向 0.6μm 极限，IPC Sensor 却"倒退"做大像素——目的不同。

BSI 背照式

Back-Side Illumination

BSI 将光电二极管放在金属布线层的上方（翻转芯片，从背面进光），消除了 FSI 中金属走线对光线的遮挡。量子效率（QE）从 FSI 的 40-50% 提升到 70-80%。星光级 Sensor 几乎全部采用 BSI 工艺。进一步有 Stacked BSI（堆栈式背照）。

AI 降噪加持

AI-NR / Deep Learning Denoise

在 0.001 lux 下即使大像素 BSI Sensor 的 SNR 也只有 ~6-10dB——噪声仍然明显。AI 降噪（CNN/Transformer）在保持纹理清晰度的前提下能额外压制 3-6dB 噪声，是微光全彩的最后一道关。但 AI-NR 需要 NPU/GPU 加速，增加功耗和成本。

微光全彩 vs 红外黑白——场景驱动的选择
红外夜视的优势是绝对照度下限低（配合 IR 补光可在 0 lux 工作），但代价是丢失色彩信息——人脸、衣服、车牌的颜色在红外下全部变成灰度。微光全彩保留了色彩，但有照度下限（0.001 lux 以下仍需 IR 辅助）。高端 IPC 的策略：优先微光全彩（保留色彩信息用于 AI 识别），只有环境光确实不足时才自动切换 IR + 白光补光混合模式——这就是"暖光全彩"或"双光融合"。

阶段 1：BLC — 黑电平校正

Black Level Correction：当没有光时，Sensor 输出也不为零——必须把这部分"暗底"减掉

1

为什么需要 BLC？

Black Level Correction

即使在完全黑暗的条件下，Sensor 的模拟电路（放大器、ADC）仍然输出一个微小的正电压——称为黑电平（Black Level）或 Optical Black。典型值在 ADC 量程的 5-10% 处（例如 12bit Sensor 的黑电平约为 64-256 ADU）。不扣除黑电平，暗区会偏灰偏紫，后续白平衡和色彩校正也会偏移。

1

BLC 的实现方式

Sensor 的边缘有几行/几列被金属遮光层覆盖——这些是Optical Black (OB) 像素。它们接受与有效像素完全相同的模拟处理（包括暗电流和电路偏置），但不受光照影响。ISP 读取 OB 像素的平均值作为黑电平参考，从所有有效像素中减去。公式很简单：P_out = P_in − BL。其中 BL 是 OB 行/列的均值（或按行/列分别计算以补偿横向条纹噪声）。

⚠️ BLC 与传感器温度的关系
暗电流随温度指数增长——温度每升高 6-8°C，暗电流翻倍。因此黑电平会随 Sensor 温度漂移。高端 ISP 会根据片上温度传感器动态调整 BLC 值，或在每帧中重新采样 OB 像素。如果 BLC 不跟踪温度变化，暗区会出现色偏或绿色条纹。

阶段 2：DPC — 坏点校正

Defect Pixel Correction：制造缺陷、宇宙射线、高温产生的异常像素必须被检测和修正

静态坏点 (Static)

制造缺陷，出厂即存在

由于硅晶圆缺陷、掺杂不均匀等原因，某些像素的暗电流异常高或响应异常低。亮坏点（Hot Pixel）在低光照下表现为白色亮点；暗坏点（Dead Pixel）完全不响应光。Sensor 厂商会提供坏点坐标表（Defect Map）。

动态坏点 (Dynamic)

使用过程中产生，需实时检测

宇宙射线、高温、老化等因素会在使用过程中产生新的坏点。ISP 需要实时检测：如果某个像素值与其邻域（同色通道）相差超过阈值，则判定为坏点，用邻域插值替代。

坏点类型检测方法校正方法复杂性
亮坏点 (Hot)与邻域同色像素比较，值异常高中值滤波 / 邻域均值替代低
暗坏点 (Dead)与邻域同色像素比较，值异常低同上低
列坏点 (Column)整列像素行为一致异常相邻列插值中

DPC 在 RAW 域的独特优势
在 Bayer 模式（每个像素只有一个颜色通道）下做坏点检测，可以利用同色通道邻域像素（例如以当前 G 像素为中心，对比上下左右的 G 像素）做出更精确的判断。如果在 RGB 域做 DPC，坏点的影响已经通过 Demosaic 扩散到三个通道，修复难度大增。

阶段 3：LSC — 镜头阴影校正

Lens Shading Correction：画面中心亮、边缘暗，且 R/G/B 通道衰减不一致——需要增益补偿

3

两种 Shading 现象

Luminance Shading（亮度阴影）：由 cos⁴ 定律导致——光线以大角度（边缘）入射时传感器接收到的光通量按 cos⁴θ 衰减。画面中心最亮，边缘亮度可下降 30-60%。
Color Shading（颜色阴影）：不同波长的光在微透镜和 CFA 中的折射角度不同，导致 R/G/B 通道的边缘衰减幅度不一样——表现为边缘偏色（通常偏青或偏品）。

3

LSC 校正方法

ISP 对每个像素位置 (x,y) 应用径向增益：P_out(x,y) = P_in(x,y) × Gain(r)，其中 r 是像素距光学中心的归一化距离。增益函数通常用 4-6 阶多项式拟合（Gain(r) = 1 + a₁r² + a₂r⁴ + a₃r⁶ + ...），R/Gr/Gb/B 四个通道分别有独立的增益曲线。校正参数在模组出厂标定中通过拍摄均匀白板得到。

LSC 过度补偿会放大噪声
边缘增益可达 2-3×，这意味着边缘像素的噪声也被放大了 2-3 倍。在低照度场景中，LSC 后的边缘 SNR 显著低于中心。解决方案：在 LSC 之前先用 RAW 域降噪，或对 LSC 增益做上限裁剪（trade-off 均匀度 vs 噪声）。
↓

阶段 4：WDR/HDR — 高动态范围合成

单次曝光只能捕获 60-70dB 动态范围——WDR 通过多次曝光将动态范围扩展到 100-140dB

DOL/WDR — 多曝光合成

Digital Overlap / Staggered HDR

Sensor 在同一帧周期内进行两次或三次曝光：短曝光 (Short) 捕获高亮区域，长曝光 (Long) 捕获暗部细节。ISP 将多帧 RAW 合成为一帧高位深（16-20bit）的线性图像。关键挑战：运动鬼影（Motion Ghost）——运动物体在长短曝光之间位置变化。

DCG — 双转换增益

Dual Conversion Gain

Sensor 像素级的技术：每个像素有两套电荷-电压转换增益——高增益 (HCG) 用于暗光（高灵敏度、低满阱）、低增益 (LCG) 用于亮光（低灵敏度、高满阱）。无需多次曝光，天然无鬼影，但动态范围增加有限（~15-20dB）。高端 Sensor 将 DOL + DCG 组合使用。

阶段 5：RAW 域降噪 — Bayer NR

在 Demosaic 之前做降噪——噪声还没有跨通道扩散，模型最干净

光子散粒噪声

Photon Shot Noise

光的量子本质——光子到达时间的随机性。服从泊松分布：信号为 S 时，噪声标准差 = sqrt(S)。信噪比 SNR = sqrt(S)——信号越强，SNR 越高。物理极限，无法消除。

读出噪声

Read Noise

像素放大器、ADC 等电路引入的固定噪声底——与信号强度无关。等效于几个到十几个电子。在暗光场景是主要噪声源。CDS（相关双采样）可以有效削减读出噪声。

模式噪声 (FPN)

Fixed Pattern Noise

由于制造工艺不均匀，每个像素的暗电流和增益略有不同——在均匀亮度的画面上呈固定的纵向条纹或颗粒纹理。列并行 ADC 还会引入列 FPN。用暗帧减法（Dark Frame Subtraction）消除。

5

Bayer 域降噪的核心思路

在 Bayer 域，一个像素只有一种颜色信息。降噪时只能在同色通道的邻域像素上操作——例如对当前的 G 像素，参考它上下左右（2像素步长）的 G 像素做加权平均。常用技术：双边滤波（Bilateral Filter）——空间域和亮度域双重加权，边缘保留效果好；NLM（Non-Local Means）——在全图搜索相似块，效果好但计算量大；AI-NR（神经网络降噪）——高端手机 ISP 的标配。

↓

阶段 6：Demosaic — 去马赛克

每个像素只有一个颜色，但我们需要 RGB 三个——Demosaic 是 ISP 中最关键、最容易出错的插值步骤

6

问题的本质

Demosaicing / Color Interpolation

Bayer 模式下，每个像素仅有 R、G、B 之一。Demosaic 需要为每个像素重建缺失的两个颜色通道——本质是一个插值问题。但简单的双线性插值会导致严重的伪影：拉链效应（Zippering）在边缘出现交替色块、伪彩色（False Color）在高频纹理区域出现彩虹纹。

双线性插值 (Bilinear)

最简单，硬件实现成本最低

对每个缺失的颜色，取邻域同色像素的平均值。G 通道用上下左右 4 邻域，R/B 通道用对角 4 邻域。计算量极小但边缘伪影严重，仅用于极低成本方案。

边缘自适应插值

AHCL / Malvar-He-Cutler

检测局部梯度方向，沿边缘方向插值（而非跨边缘）。先插 G 通道（因为 G 采样率最高），再以色差恒定假设（R-G、B-G 变化平缓）插 R 和 B。这是硬件 ISP 的主流方法，性价比最优。

RES / AHD 高级方法

Residual Interpolation / Adaptive Homogeneity-Directed

在残差域（R-G 或 log 域）做插值，更符合色差平滑假设。AHD 在同质区域用双线性、在边缘用梯度导向插值。伪影更少，但需要更大的计算和行缓存。

AI/Deep Learning Demosaic

神经网络方法

CNN/Transformer 直接学习 Bayer→RGB 的映射。表现最好（高频纹理、摩尔纹抑制），但需要 NPU/GPU 加速且功耗高。高端手机（Apple Deep Fusion、Qualcomm Spectra）的标配。

阶段 7-8：AWB + CCM — 白平衡与色彩校正

从"物理颜色"到"感知颜色"——让白色在任意光源下都呈现为白色，再让 RGB 接近人眼

7

AWB (Auto White Balance) 自动白平衡

R_gain × R, G_gain × G, B_gain × B

不同光源有不同的色温：烛光 ~2000K（偏橙）、白炽灯 ~3200K（偏黄）、日光 ~5500K（白）、阴天 ~6500K（偏蓝）。AWB 的任务是估计场景光源色温，计算 R/G/B 增益使白色物体在画面上呈现为 R=G=B。

灰世界法 (Gray World)

假设场景的平均颜色是灰色

计算整帧 R、G、B 的均值，令 G 增益为 1，调整 R/B 增益使三者均值相等。简单高效，但大色块场景（大片绿草地、红叶）会严重偏色。

白点检测法

只检测画面中的"近白点"

在色温曲线（Planckian Locus）附近搜索可能是白色的像素（R/G≈B/G≈1 的区域），只对这些像素做灰世界计算。抗大色块干扰能力强，主流 ISP 的标准做法。

8

CCM (Color Correction Matrix) 色彩校正矩阵

3×3 矩阵：将 Sensor RGB 变换到标准 sRGB

AWB 只解决了白平衡，但 Sensor 的光谱响应曲线与人眼的三色刺激函数（CIE 1931）不完全匹配——也就是说 Sensor 看到的"红"和人眼看到的"红"不太一样。CCM 用一个 3×3 矩阵（或更高阶的 3×n 多项式）将 Sensor 的 RGB 映射到标准色彩空间。参数通过拍摄标准色卡（X-Rite ColorChecker 24 色）最小化色差（ΔE）拟合得到。

色彩饱和度和"记忆色"偏好
严格色彩还原（Colorimetric）不一定好看——人脑对蓝天、绿草、肤色有"记忆色"偏好。ISP 通常在 CCM 之后还会叠加色相映射（Hue Mapping）和饱和度增强（Saturation Boost），让天空更蓝、肤色更粉嫩。这就是为什么不同手机拍同一场景颜色不一样——差异不在传感器，而在色彩调校哲学。
↓

阶段 9：Gamma 校正与色调映射

线性光的巨大动态范围需要被压缩到显示器的非线性空间——Gamma 是连接二者的桥梁

9

为什么需要 Gamma 校正？

Gamma Correction = 编码端的非线性映射

人眼对暗部亮度变化的敏感度远高于亮部（Weber-Fechner 定律）。线性编码中，50% 亮度用掉一半码字，但人眼很难区分 50% 和 51% 的亮度差——这是巨大的编码效率浪费。Gamma 校正将更多码字分配给暗部，匹配人眼的感知特性。同时，CRT/LCD 显示器天然的电压-亮度响应是 Gamma 2.2，需要端到端 Gamma = 1.0（编码 Gamma × 显示 Gamma ≈ 1.0）。

sRGB Gamma

≈ Gamma 2.2

最广泛的标准。线性段（暗部）+ 幂函数段（中高亮）。公式：暗部线性（斜率≈12.92），亮部 power(1/2.4)。Web、PC、消费级显示的默认色彩空间。

Rec.709 (BT.709)

≈ Gamma 2.4

HDTV/1080p 标准。Gamma 略高于 sRGB，比 sRGB 稍暗。配合 Rec.709 色域（与 sRGB 色域相同），是 H.264 编码器最常用的输入色彩空间。

PQ / HLG (HDR)

Perceptual Quantizer / Hybrid Log-Gamma

PQ (ST.2084)：基于人眼绝对亮度感知阈值设计，支持 0-10000 nit。HDR10 / Dolby Vision 使用。
HLG：下半段是 Gamma 曲线（兼容 SDR 显示器），上半段是对数曲线（HDR 信息）。广播电视 HDR 方案。

局部色调映射 (LTM / Local Tone Mapping)
全局 Gamma 曲线的问题是：画面的亮区和暗区共享同一曲线——照顾暗部时亮部过曝，照顾亮部时暗部细节丢失。LTM 将画面分割成块，每块根据其局部亮度自适应调整映射曲线：暗块提亮、亮块压低，大幅提升动态范围的视觉利用率。代价是：块边界可能出现光晕（Halo），全局对比度可能降低——需要精心调校。
↓

阶段 10：色彩空间转换 — RGB → YUV

Gamma-encoded RGB 转为 YUV 后，亮度与色彩分离——为后续独立降噪、色度下采样和编码做准备

10

RGB → YUV 的本质

YUV 将图像分解为：Y（Luma，亮度）——携带结构、边缘、纹理等感知敏感信息；U/V（Chroma，色度）——携带颜色信息，人眼对色度分辨率不敏感。转换公式（BT.601 标准）：
Y = 0.299R + 0.587G + 0.114B
Cb = 0.564(B − Y)  Cr = 0.713(R − Y)

阶段 11-12：YUV 域降噪 — 2D/3D NR

RAW 域降噪后再在 YUV 域做精细化降噪——亮度和色度分开处理，利用时域信息

2D NR — 空间域降噪

2D Spatial Noise Reduction

在当前帧内部进行降噪。Y 通道和 UV 通道分开处理——Y 通道轻手降噪（保留纹理细节），UV 通道重手降噪（色度噪声人眼敏感但色度细节不敏感）。常用 Bilateral Filter、Guided Filter、小波阈值降噪等。不会引入拖影（因为是单帧处理），但降噪强度受限。

3D NR — 时域降噪

3D Temporal Noise Reduction

利用连续多帧的平均来消除随机噪声——因为噪声是随机的（帧间无关），而信号是稳定的（帧间相关）。核心是运动自适应/运动补偿的时域滤波：静止区域做多帧强滤波 → SNR 大幅提升；运动区域做运动补偿对齐后再滤波 → 避免拖影。3D NR 的降噪效果远超 2D NR，但运动检测和鬼影问题是难点。

YUV 降噪的最佳实践：分层策略
ISP 降噪通常分三层：RAW 域（Bayer NR，最小化跨通道扩散）→ RGB 域（Demosaic 后残留噪声）→ YUV 域（Y/UV 分开，2D+3D 组合）。每一层只处理自己最擅长的噪声类型。把全部降噪堆在 YUV 域会损失细节；把全部降噪堆在 RAW 域则无法利用跨通道和时域信息。

阶段 13：锐化 — Sharpening

降噪让画面干净但也变软——锐化是补偿过程，但做过头就是振铃和噪点增强

13

Unsharp Mask (USM) — 最经典的锐化方法

Sharp = Original + Amount × (Original − Blurred)

1. 对原图做高斯模糊 → 2. 原图减去模糊图得到细节层（高频分量）→ 3. 原图 + Gain × 细节层 = 锐化输出。三个参数：Amount（增益强度）、Radius（影响多大尺度的边缘）、Threshold（低于此值的细节不增强，避免放大平坦区噪声）。

13

边缘增强 vs 纹理增强

高端 ISP 将锐化分为两个独立模块：Edge Enhance（边缘增强）针对大振幅的强边缘（如物体轮廓），使用较大的 Radius 和较小的 Amount，产生清晰分界感；Texture Enhance（纹理增强）针对小振幅的弱纹理（如皮肤毛孔、织物纹理），使用较小的 Radius 和适当的 Amount，增加画面细腻度。两者独立可调避免互相干扰。

锐化的代价：振铃效应与噪声放大
锐化本质是强调高频分量——但噪声也是高频。过度锐化会在强边缘两侧产生光晕（Halo/Ringing）（USM 的 Radius 越大越明显），同时在平坦区域放大噪声。好的 ISP 会将降噪和锐化设计为互补的频带分割策略：降噪处理高频噪声段，锐化处理中高频信号段，两者不重叠。
↓

LDC：镜头畸变校正

广角镜头带来大视角，也带来了桶形畸变——ISP 需要用几何变换把弯曲的直线"拉直"

LDC

畸变的类型与成因

Lens Distortion Correction — 几何变换纠正镜头畸变

桶形畸变（Barrel）：广角镜头最常见——画面中心的放大率大于边缘，直线向外弯曲。IPC 超广角（FOV≥120°）非常明显。
枕形畸变（Pincushion）：长焦镜头常见——中心放大率小于边缘，直线向内弯。
胡子畸变（Mustache）：桶形+枕形混合——近距离桶形、远距离枕形，最复杂。
畸变的根源是镜片组的光学设计——完全消除畸变需要更多镜片（成本、体积、透光率），因此用 ISP 做软件校正更经济。

径向畸变模型

Brown-Conrady 多项式模型

用归一化坐标 (x,y) 描述畸变：
x' = x(1 + k₁r² + k₂r⁴ + k₃r⁶)
y' = y(1 + k₁r² + k₂r⁴ + k₃r⁶)
其中 r² = x² + y²（距光心的距离），k₁/k₂/k₃ 是畸变系数。桶形畸变时 k₁ 为负，枕形畸变为正。通常 3 个系数就足够，极端鱼眼需要 5-6 个。

切向畸变模型

镜头组装偏心引入

镜头与 Sensor 平面不完全平行导致：
x' = x + [2p₁xy + p₂(r²+2x²)]
y' = y + [p₁(r²+2y²) + 2p₂xy]
p₁/p₂ 通常很小（~10⁻⁴ 量级），但在高像素 Sensor 上仍可察觉。高端模组出厂前会逐颗标定畸变参数并烧录到 OTP 中。

HW

LDC 的硬件实现与管线位置

Warp Engine / DMA-based Geometric Transformation

LDC 本质是对图像做逐像素重映射（Remap）——校正后的每个输出像素，通过畸变模型反算其在输入图中的位置，再用插值（双线性/Bicubic）取色。这需要随机访存（不像其他 ISP 阶段是顺序处理），因此很多 ISP 用专用 Warp DMA 引擎做高效映射表查表（LUT）。管线位置通常在 YUV 域——色彩已重建完毕，几何变换不会引入新的颜色伪影。对于超广角（≥140°），LDC 通常与鱼眼展开（Dewarping）合并处理。

畸变校正的代价——分辨率的空间不均匀损失
桶形校正时边缘被拉伸（需要插值放大），而 Sensor 的边缘像素本来在镜头 MTF 衰减下就已经比中心模糊——LDC 边缘拉伸会进一步放大边缘模糊。结果：校正后画面中心清晰、边缘软。缓解方法：① LDC 在校正量上做 trade-off（不完全拉直，留一点畸变换取更好的边缘解析力）；② 使用更高分辨率的 Sensor 给 LDC 留出"裁剪余量"；③ 结合电子防抖（EIS）——LDC 裁剪掉边缘区域刚好给 EIS 提供位移空间。
↓

阶段 14-15：缩放与输出格式

ISP 输出需要缩放到目标分辨率，并封装为编码器友好的像素格式

缩放 (Scaling)

Downscale / Upscale

ISP 通常输出多个分辨率的流：主路（Main Stream） 1080p/4K 供编码录像、子路（Sub Stream） 640×480 供预览/AI 检测、抓图流（Snapshot） 全分辨率供拍照。缩放滤波器：双线性（快但模糊）、Lanczos（锐利但振铃）、Bicubic（折中选择）。

像素格式转换

YUV444 → YUV422 → YUV420/NV12

硬件编码器通常要求 NV12（Y 平面 + UV 交错平面）或 NV21（Y 平面 + VU 交错平面）。ISP 从 YUV444 下采样到 YUV420：UV 平面的水平和垂直分辨率各降一半。色度下采样的位置（cositing）需要与编码器的期望一致，否则会出现色度偏移。

3A 算法：AE / AF / AWB

3A 是 ISP 的闭环控制核心——根据图像统计数据，实时调整 Sensor、镜头和 ISP 参数

AE

AE — Auto Exposure 自动曝光

控制曝光时间 + 模拟增益 + 数字增益 + 光圈

AE 计算画面亮度直方图，与目标亮度（Target Luma）比较，调整四条"曝光杠杆"：1) 曝光时间（微秒到毫秒级，运动模糊 vs 暗光 SNR）、2) 模拟增益（放大器增益，1×~16×，噪声友好但上限有限）、3) 数字增益（ISP 数字放大，噪声不友好仅限于救急）、4) 光圈（硬件调整，普通相机不用）。AE 收敛速度通常在 3-10 帧，收敛太快会振荡，太慢则延迟明显。

AF

AF — Auto Focus 自动对焦

对比度检测 (CDAF) / 相位检测 (PDAF) / 激光辅助

CDAF（反差对焦）：通过移动镜头，计算 ROI 区域对比度（高频能量），搜索对比度峰值。准确但慢（需要反复移动镜头）。
PDAF（相位对焦）：Sensor 上部分像素被遮挡（一半遮左一半遮右），通过左右像素的信号相位差直接计算离焦量和方向——一次到位，速度快。高端手机和微单标配。
激光对焦：ToF 测距辅助近距离快速对焦。

AWB

AWB — Auto White Balance 自动白平衡

光源色温估计 + R/G/B 增益计算

在 S13 已详细描述。补充闭环控制逻辑：AWB 根据当前帧统计量计算 R/G/B 增益 → 应用到下一帧 → 再统计验证白点分布 → 迭代收敛。为防止色温跳变引起的画面闪烁，AWB 增益变化需要阻尼（Damping）——每次调整不超过 5-10%，过渡平滑。

AE 与 AWB 的耦合问题
曝光改变亮度 → 影响 AWB 的白点检测（白点判定依赖于像素亮度） → AWB 改变增益 → 影响画面亮度 → 影响 AE 判断。如果两者独立运行，容易产生振荡。好的 3A 设计需要AE-AWB 联合控制：AE 先收敛 → 锁定 → AWB 再收敛 → 解锁 → AE 微调。或使用收敛状态机协调两者的更新时序。

ISP 硬件架构：管线、缓存与带宽

ISP 的高吞吐处理依赖于精心设计的流水线架构和 DDR 带宽管理

硬线 ISP (Hardwired Pipeline)

固定功能流水线，ASIC 实现

每个处理阶段都是专用的硬件模块，数据从前一站直通下一站。功耗极低、延迟极小（2-5 行缓存即可启动处理），但算法固定、不可编程。嵌入式 SoC（海思、安霸、Ingenic）和低功耗 Camera 芯片的标配。

可编程 ISP

SIMD/DSP + 部分硬线

核心阶段（Demosaic、NR）用可编程 SIMD 向量处理器实现，轻量阶段（BLC、Gamma）保留硬线。兼顾灵活性和功耗。Apple ISP、Qualcomm Spectra 都采用此架构，允许通过固件升级改善画质。

架构类型代表方案灵活性功耗延迟成本适用场景

ISP 的 DDR 带宽需求——4K 场景下的真实压力
ISP 处理过程中需要大量读写 DDR（特别是 WDR 多帧合成和 3D NR 需要缓存多帧）。以 4K@30fps YUV420 为例：
• 读 RAW (12bit Bayer): 3840×2160×12/8 ≈ 12.4 MB/帧 → 373 MB/s
• 写多路 YUV 输出 (主+子): ~15 MB/帧 → 450 MB/s
• 3D NR 前后帧读写: ×2~3 → 总带宽可轻易突破 1.5-2 GB/s
这对 DDR3/DDR4 控制器和系统总线（AXI）是巨大的考验，ISP 通常有专用的多级缓存（Line Buffer / Tile Buffer）来降低片外访问次数。

ISP 调校实践

一个好的 ISP 参数配置需要数周甚至数月的拍摄-分析-调优循环

1. 黑电平标定

暗室 + OB 像素统计

在全黑环境下采集多帧，统计 OB 像素的均值、标准差和行/列剖面。确定 BLC 基准值和行噪声补偿参数。需在不同温度和增益下重复。

2. LSC 标定

均匀光源 + 多色温

拍摄均匀白板（积分球/灯箱），在 D65/D50/A 等多个色温下分别采集，拟合每个色温下 R/G/B 四通道的径向衰减曲线。LSC 应在 BLC 之后标定。

3. CCM 标定

24 色卡 + 最小二乘拟合

拍摄标准色卡（X-Rite ColorChecker），提取每个色块的 Sensor RGB 值，用最小二乘法拟合 3×3 CCM 矩阵使 ΔE 最小化。需要分色温做多组 CCM并随 AWB 插值。

4. 降噪参数

多 ISO 级别调节

在不同增益（ISO 100/400/1600/6400）下拍摄灰卡和纹理图，平衡噪声抑制与细节保留。Y 通道和 UV 通道独立调优。3D NR 的运动阈值需要运动场景验证。

5. AE 策略

目标亮度 + 曝光表

设定目标亮度（Target Luma，通常 18% 灰 = ~128 ADU），编制曝光表（Exposure Table）——在每条亮度水平下，曝光时间和增益的最优组合。一般优先增加曝光时间（直到运动模糊极限），再增加模拟增益（直到设定的最高增益）。

6. 主观调优

记忆色 + 场景偏好

仪器标定只能保证色彩还原准确——但"好看"和"准确"不是一回事。最终调校需要：肤色偏好（白种人/黄种人/黑人不同）、天空蓝（饱和但不过饱和）、绿植（鲜活但不荧光）。需要大量实景拍摄和主观评价迭代。

IQ Tuning 工具链
工业级 ISP 调校依赖专业工具：Imatest / DXO Analyzer（客观指标测试——MTF、噪声、色差、动态范围）、灯箱/标准光源（D65/D50/TL84/A/Horizon 等多种光源）、透射式色卡（精确控制曝光量）、自动调优框架（海思 PQTool、安霸 AFT——半自动搜索最优参数组合）。

IPC 实战：经验教训与工程陷阱

实验室里调好的 ISP 参数，到了真实场景往往一塌糊涂——这是 IPC 产品化最痛的领悟

1

日夜切换的工程细节

Day/Night Switch — 不止是 IR-Cut 切换那么简单

日夜切换涉及一连串联动操作：① ICR 电磁铁动作（200-500ms）→ ② ISP 切换日夜两组参数（AWB 目标、Gamma 曲线、降噪强度、AE 目标亮度都不同）→ ③ 如果从白天切到夜晚且 IR LED 开启，AE 需要重新收敛（IR 补光改变了整个场景的亮度分布）→ ④ 从夜切回白天时，IR LED 需要先关再切 ICR（否则 Sensor 在无 IR-Cut 下被 IR LED 照射的画面会过曝）。整套切换序列的时序编排如果出错，就会出现闪烁、偏色、过曝。经验：用状态机严格管理切换流程，每个步骤设置超时和回退机制。

电源噪声 → 图像横纹

DC-DC 纹波耦合到 Sensor AVDD

Sensor 的模拟供电（AVDD 2.8V）对噪声极度敏感。劣质 DC-DC 的开关纹波（100-500kHz）耦合到 AVDD 上，会表现为画面固定频率的横向条纹（Banding）——尤其在低照度高增益时明显。排查：用示波器量 AVDD 纹波，目标 <10mVpp。解决：加 LDO 二次稳压（DC-DC → LDO → AVDD），或增大去耦电容 + 磁珠滤波。

MIPI 信号完整性

长排线 + 高速差分信号 = 灾难配方

MIPI CSI 每 Lane 速率 1-2.5 Gbps——这是射频级别的信号。排线超过 10cm、FPC 弯折、连接器阻抗不匹配都可能导致：花屏（CRC 错误不纠错）、间歇性丢帧、特定分辨率/帧率下才出问题。实测经验：排线 ≤ 5cm 安全、5-15cm 需要等长设计和阻抗控制、≥15cm 需用同轴线或 GMSL/FPD-Link 转换。

温度对焦点的影响

热胀冷缩导致 Focus Shift

IPC 在户外经历 -20°C 到 60°C 的温差——镜筒金属材料的热胀冷缩（~20μm 量级）足以让焦点偏移。白天可能清晰，夜晚低温下变模糊。高端 IPC 镜头使用低热膨胀系数材料（如钛合金镜筒），或在 ISP 固件中根据温度传感器做 AF 补偿。定焦镜头出厂时需在不同温度下标定最佳焦面。

球机护罩与光学污染

Dome Bubble 的 AR 镀膜退化

球机透明护罩（PC/PMMA 材质）在户外紫外线照射下逐渐黄化，AR 镀膜也会被风沙侵蚀——2-3 年后画面反差下降、杂散光增加。护罩内表面凝露/起雾是最头疼的问题：昼夜温差导致内部水汽凝结。解决：填充氮气密封、放置干燥剂包、加热除雾（加热器 + 温度传感器自动控制）。

IR 补光的实际痛点

反射、过曝、角度匹配

• IR LED 发光角度必须与镜头 FOV 匹配——角度太小则画面边缘暗（手电筒效应）、太大则近处过曝远处不足
• 球机/半球 IR 灯板与镜头在同一腔体内，护罩内反射造成白雾/光晕——灯板与镜头之间必须有不透光隔离结构
• 蜘蛛喜欢在 IR LED 附近结网（红外光吸引昆虫→吸引蜘蛛）——定期清洁或物理防虫设计

Sensor 老化与像素退化

长期运行后的画质漂移

• 热像素（Hot Pixel）增加：Sensor 运行数年后暗电流增大，新出现亮坏点——DPC 无法全部覆盖 → 需定期更新坏点表
• CFA 褪色：紫外线长期照射使颜色滤光片染料退化 → 色彩饱和度下降、白平衡漂移 → 需定期重新标定 CCM/AWB
• BLC 漂移：暗电流随工作年限增大 → 黑电平基准偏移 → 暗区偏色。需在 OTA 固件中更新 BLC 参数

🔧 IPC 现场调试的黄金法则
1. 永远怀疑电源。图像异常的第一检查项不是 ISP 参数、不是 Sensor 寄存器——是用示波器看 AVDD/DVDD/DOVDD 纹波。70% 的"ISP 问题"其实是模拟供电噪声。
2. AWB 是大色块场景的第一受害者。绿草地、红砖墙、大面积单色物体——Gray World 假设直接崩溃。如果客户投诉"白天偏色"，先检查画面中是否有大面积单色区域。
3. 3D NR 的拖影只在运动场景暴露。实验室用静态图测 3D NR 看不出问题——必须用实际运动场景（行人、车辆）验证运动鬼影。运动阈值设太高 → 拖影；设太低 → 降噪效果不足。
4. ISP 参数不能"一套走天下"。室内/室外、白天/夜晚、晴天/阴天——至少准备 4-6 组场景参数。好的 3A 库会根据 AE 亮度 + AWB 色温 + 运动检测自动切换场景 Profile。

总结：ISP 选型与性能关键路径

回顾整个 Pipeline，记住几个关键决策点

Sensor 选型四要素

像素尺寸 · 快门类型 · 动态范围 · 接口

• 暗光优先：大像素 (≥2.0μm) + BSI + DCG
• 高速运动：Global Shutter 或极快读出
• 高动态场景：DOL WDR ≥120dB
• 接口：4K+ 用 MIPI 4-lane，1080p 用 2-lane

ISP 选型要点

3A 能力 · 降噪 · WDR · 编码集成

• 3A 智能程度：复杂光源 AWB、混合 AF 最见功力
• 降噪水平：3D NR + AI NR 拉开高低端差距
• WDR 合成：de-ghost 能力是关键
• 集成编码器：ISP→编码 DMA 直通降低延迟

性能关键路径 (Latency)

光子到编码输出的端到端延迟

• 曝光时间：1-33ms（取决于帧率和光照）
• Sensor 读出：5-20ms（取决于分辨率和 MIPI 速率）
• ISP Pipeline：1-3 帧（33-100ms @30fps）
• 编码 (H.264 I 帧)：5-15ms @1080p
• 关键词：减少 DDR 往返、减少帧缓存、使用行级流式处理

画质关键路径 (Image Quality)

决定最终画质差异的核心环节

• 光学 > Sensor > ISP > 编码——源头决定上限
• ISP 内部：Demosaic > 降噪 > Tone Mapping > 锐化
• 好的光学 + 中端 ISP > 差的光学 + 旗舰 ISP
• 好的 Tuning > 好的硬件（但差硬件做不了好 Tuning）

🔬 光学→ 📷 Sensor→ ⚙️ ISP→ 🎞️ 编码器→ 📡 传输 / 存储

每一步都在为下一步"打基础"——前一步的信息丢失永远无法在后一步恢复
